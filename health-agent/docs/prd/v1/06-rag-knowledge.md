# V1 - RAG 知识库与工具调用

> 模块：健康建议知识库（RAG）+ 食物营养查询（Tool Use）
> 优先级：P0（第一批开发，可与用户体系并行）
> 依赖：无
> 预计工时：1.5-2 周

---

## 1. 模块目标

构建健康建议知识库（RAG）和食物营养查询工具（Tool Use），为 AI 建议和饮食记录提供准确的知识支撑。

### 1.1 核心目标
- 提供专业的健康建议知识（RAG）
- 提供准确的食物营养数据（Tool Use）
- 让 AI 建议更准确、更可信
- 降低 Token 消耗

### 1.2 V1 标准
- 健康建议库覆盖基础场景（180 条知识）
- 食物数据库覆盖 500-1000 种常见食物
- 不追求医学级专业知识和完整食物百科

### 1.3 架构设计

| 数据类型 | 实现方式 | 存储 | 原因 |
|---------|---------|------|------|
| 健康建议知识 | RAG（向量检索） | Supabase pgvector + PostgreSQL | 语义匹配、专业知识、可控性 |
| 食物营养数据 | Tool Use（工具调用） | PostgreSQL | 精确查询、低 token、实现简单 |

## 2. 健康建议知识库（RAG）

### 2.1 知识库内容

#### 2.1.1 基础营养知识
- 三大营养素的作用和推荐摄入量
- 常见营养素缺乏的症状和建议
- 不同目标（减脂/增肌/保持）的营养配比
- 不同活动量的热量需求

#### 2.1.2 饮食建议
- 减脂期饮食建议
- 增肌期饮食建议
- 控制慢性病的饮食建议（高血压、糖尿病、痛风等）
- 不同人群的特殊饮食需求

#### 2.1.3 食物搭配知识
- 优质蛋白质来源
- 低 GI 食物推荐
- 高纤维食物推荐
- 营养素协同作用

#### 2.1.4 常见问题解答
- "减脂期能吃水果吗？"
- "晚上吃碳水会胖吗？"
- "蛋白质吃多了有害吗？"
- "如何判断自己是否缺乏某种营养素？"

### 2.2 知识组织方式

#### 2.2.1 按场景组织
- 饮食记录场景：营养分析、食物建议
- 计划制定场景：目标相关的营养知识
- 数据分析场景：趋势解读、改进建议

#### 2.2.2 按用户特征组织
- 减脂用户：低热量、高蛋白、低 GI 食物
- 增肌用户：高蛋白、适量碳水
- 慢性病用户：疾病相关的饮食限制

### 2.3 知识文档结构

```json
{
  "id": "uuid",
  "title": "减脂期蛋白质摄入建议",
  "content": "减脂期建议每日蛋白质摄入量为每公斤体重 1.6-2.2g。优质蛋白质来源包括：鸡胸肉、鱼肉、豆腐、鸡蛋等。蛋白质可以增加饱腹感，减少肌肉流失...",
  "embedding": [0.1, 0.2, ...],
  "metadata": {
    "category": "减脂建议",
    "tags": ["蛋白质", "减脂", "食物推荐"],
    "target_users": ["减脂用户"],
    "source": "中国营养学会推荐"
  },

```

### 2.4 检索方式

**向量检索流程：**
```
用户问题："我今天蛋白质摄入偏低，应该吃什么？"
    ↓
Embedding 向量化
    ↓
Supabase pgvector 向量检索（Top 3-5）
    ↓
返回相关知识片段：
  1. 优质蛋白质来源：鸡胸肉、鱼肉、豆腐...
  2. 蛋白质推荐摄入量：每公斤体重 1.2-2.0g
  3. 减脂期蛋白质的重要性：...
    ↓
组装成 LLM 上下文
    ↓
LLM 生成个性化建议
```

## 3. 食物营养查询（Tool Use）

### 3.1 工具定义

```python
tools = [
    {
        "name": "query_food_nutrition",
        "description": "查询食物的营养成分数据",
        "parameters": {
            "type": "object",
            "properties": {
                "food_name": {
                    "type": "string",
                    "description": "食物名称，如'鸡胸肉'、'米饭'"
                },
                "amount": {
                    "type": "number",
                    "description": "份量（克），默认 100g"
                }
            },
            "required": ["food_name"]
        }
    }
]
```

### 3.2 数据源策略（三级回退）

```
1. 本地数据库（优先）
   ↓ 未命中
2. 第三方 API（FatSecret / Open Food Facts）
   ↓ 未命中
3. LLM 估算（兜底，标记"AI 估算"）
```

### 3.3 本地食物数据库

#### 3.3.1 数据结构

```sql
CREATE TABLE foods (
  id UUID PRIMARY KEY,
  name VARCHAR(100) NOT NULL,
  aliases TEXT[], -- 别名，如 ["土豆", "马铃薯"]
  category VARCHAR(50), -- 分类：主食、肉类、蔬菜等
  calories DECIMAL(8,2), -- 每 100g 热量 kcal
  protein DECIMAL(8,2), -- 蛋白质 g
  fat DECIMAL(8,2), -- 脂肪 g
  carbs DECIMAL(8,2), -- 碳水化合物 g
  fiber DECIMAL(8,2), -- 膳食纤维 g
  sodium DECIMAL(8,2), -- 钠 mg
  common_portions JSONB, -- 常见份量换算
  data_source VARCHAR(50), -- 数据来源
  created_at TIMESTAMP,
  updated_at TIMESTAMP
);

-- 常见份量换算示例
common_portions: {
  "一碗": 200,
  "一盘": 300,
  "一份": 150
}
```

| 肉类 | 80 种 | 猪牛羊鸡鱼及各部位 |

#### 3.3.3 数据来源

**V1 数据来源优先级：**

2. 手动录入常见食物
3. 第三方 API 导入

### 3.4 工具实现逻辑

```python
def query_food_nutrition(food_name: str, amount: float = 100):
    """
    查询食物营养数据

    返回格式：
    {
        "name": "鸡胸肉",
        "amount": 100,
        "unit": "g",
        "calories": 165,
        "protein": 31,
        "fat": 3.6,
        "carbs": 0,
        "data_source": "local_db"
    }
    """
    # 1. 本地数据库查询（精确匹配 + 别名匹配）
    result = db.query(
        "SELECT * FROM foods WHERE name = ? OR ? = ANY(aliases)",
        food_name, food_name
    )
    if result:
        return scale_nutrition(result, amount)

    # 2. 第三方 API 查询
    result = fatsecret_api.search(food_name)
    if result:
        # 保存到本地数据库
        db.insert_food(result)
        return scale_nutrition(result, amount)

    # 3. LLM 估算（兜底）
    result = llm_estimate_nutrition(food_name, amount)
    result["data_source"] = "ai_estimate"
    result["note"] = "AI 估算，仅供参考"
    return result
```

### 3.5 查询示例

**场景 1：本地数据库命中**
```

    ↓
LLM 调用工具：query_food_nutrition(food_name="鸡胸肉")
    ↓
本地数据库查询 → 命中
    ↓
返回：
{
  "name": "鸡胸肉",
  "amount": 100,
  "unit": "g",
  "calories": 165,
  "protein": 31,
  "fat": 3.6,

}

```

    ↓

    ↓

  "note": "AI 估算，仅供参考"
}

## 4. 使用场景

### 4.1 饮食记录场景

```
用户输入："今天吃了鸡胸肉"
    ↓
LLM 解析意图 → record_diet
    ↓
LLM 调用工具：query_food_nutrition(food_name="鸡胸肉", amount=100)
    ↓
返回营养数据
    ↓

```

### 4.2 营养查询场景

```
用户输入："鸡胸肉的热量是多少"
    ↓
LLM 解析意图 → query_nutrition
    ↓
LLM 调用工具：query_food_nutrition(food_name="鸡胸肉")
    ↓
返回营养数据
    ↓
LLM 生成回答（学习模式下附带知识讲解）
```

```
系统分析用户今日饮食
    ↓
发现蛋白质摄入偏低
    ↓
从健康建议知识库检索（RAG）：
  "优质蛋白质来源：鸡胸肉、鱼肉、豆腐..."

LLM 生成建议："建议晚餐补充鸡胸肉或鱼肉"
```

### 4.4 计划制定场景

```
用户创建减脂计划
    ↓
从健康建议知识库检索（RAG）：

```

## 5. 数据质量标准

### 5.1 食物营养数据
- 热量数据误差 < 5%
- 三大营养素数据误差 < 10%
- 常见份量换算合理（参考实际情况）
- 数据来源标记清晰

### 5.2 健康建议
- 基于科学依据，不传播伪科学
- 避免绝对化表述（如"绝对不能吃"）
- 标注适用人群和限制条件
- 引用权威来源

### 5.3 查询准确性
- 本地数据库命中率 > 70%
- 第三方 API 补充命中率 > 20%
- LLM 估算使用率 < 10%
- 工具调用成功率 > 95%

## 6. 知识库管理

### 6.1 V1 管理方式
- 手动维护（开发者直接操作数据库）
- 不做管理后台

### 6.2 数据更新机制

- 用户查询未命中的食物，记录到待补充列表
- 定期批量补充（每周一次）
- 用户纠正数据后更新数据库

**健康建议库：**
- 定期审核和更新（每月一次）
- 根据用户反馈调整内容

# V1 - 身体数据与辅助记录

> 模块：身体数据追踪（体重、围度、辅助健康记录）
> 优先级：P1（第三批开发）
> 依赖：02-应用框架
> 预计工时：1.5-2 周

---

## 1. 模块目标

记录用户的身体数据和辅助健康数据，提供趋势可视化，为计划追踪和 AI 建议提供数据支撑。

### 1.1 核心目标
- 用户能记录体重并查看趋势
- 用户能记录围度（腰围、臀围、大腿围）并查看趋势
- 用户能记录辅助健康数据（睡眠、运动、饮水、排便）
- 提供基础趋势图表
- 支持 AI 对话式记录和修改

### 1.2 V1 标准
- 支持手动录入和 AI 对话式录入
- 趋势图覆盖 7 天 / 30 天 / 90 天 / 365 天
- 辅助记录做简单版，不追求复杂分析
- 页面内卡片交互，与饮食记录保持一致

## 2. 功能清单

### 2.1 体重记录

#### 2.1.1 记录方式

**方式 1：AI 对话式记录**
- 在 AI 输入框输入："今天体重 74.5kg"
- AI 解析后更新体重卡片
- 卡片显示 [✓ 确认] [✏️ 修改] [✗ 取消] 按钮

**方式 2：手动录入**
- 点击"记录体重"按钮
- 弹出表单弹窗
- 填写后保存

#### 2.1.2 记录字段

| 字段 | 类型 | 说明 |
|------|------|------|
| 日期 | 日期 | 默认今天，可选历史日期 |
| 体重 | 数字 | 单位 kg，精确到 0.1 |
| 备注 | 文本 | 可选，如"空腹称重" |

#### 2.1.3 体重趋势图
- 折线图展示体重变化
- 时间范围：7 天 / 30 天 / 90 天 / 365 天
- 标注目标体重线
- 显示最高/最低/平均值

#### 2.1.4 页面内卡片交互

```
数据页面 - 体重 Tab：
┌─────────────────────────────┐
│ 体重趋势图（30天）           │
│ [7天] [30天] [90天] [365天] │
├─────────────────────────────┤
│ 今日体重 (待确认)            │
│ ┌─────────────────────────┐ │
│ │ 74.5 kg                 │ │
│ │ 比昨天 +0.3kg           │ │
│ │                         │ │
│ │ [✓ 确认] [✏️ 修改] [✗ 取消] │ │
│ └─────────────────────────┘ │
│                             │
│ 历史记录                    │
│ 2026-04-26  74.2kg         │
│ 2026-04-25  74.0kg         │
├─────────────────────────────┤
│ [📷] [🎤] 说点什么...  [发送] │
└─────────────────────────────┘
```

**交互流程：**
```
用户在输入框输入："今天体重 74.5kg"
    ↓
AI 解析 → 识别为体重记录
    ↓
今日体重卡片状态变为 pending
    ↓
卡片内容更新：显示"74.5 kg"
    ↓
卡片底部出现：[✓ 确认] [✏️ 修改] [✗ 取消]
    ↓
用户点击 [✓ 确认] → 保存，卡片状态变为 recorded
```

#### 2.1.5 修改记录的两种方式

**方式 1：详情页修改（表单式）**

```
用户点击卡片上的 [✏️ 修改]
    ↓
跳转到详情页 /body/weight/{id}/edit
    ↓
表单预填充当前记录：
  - 日期：[2026-04-27]
  - 体重：[74.5] kg
  - 备注：[空腹称重]
    ↓
用户修改后点 [保存]
    ↓
返回数据页面，卡片更新
```

**方式 2：对话式修改（AI 理解）**

```
用户在输入框说："把今天的体重改成 74.2kg"
    ↓
AI 解析意图：
{
  "intent": "modify_weight",
  "target": {"date": "today"},
  "change": {"weight": 74.2}
}
    ↓
前端定位到今日体重卡片
    ↓
更新卡片内容：74.2 kg
    ↓
卡片重新显示 [✓ 确认] [✏️ 修改] [✗ 取消]
```

**支持的修改指令：**
- 修改体重："把今天的体重改成 74.2kg"
- 修改备注："今天体重备注改成空腹称重"
- 删除记录："删除今天的体重记录"

### 2.3 辅助健康记录

#### 2.3.1 睡眠记录

**记录方式：**
- AI 对话式："昨晚 11 点睡，今早 7 点起"
- 手动录入：点击"记录睡眠"按钮

**字段：**

| 字段 | 类型 | 说明 |
|------|------|------|
| 日期 | 日期 | 默认今天 |
| 入睡时间 | 时间 | 如 23:00 |
| 起床时间 | 时间 | 如 07:00 |
| 睡眠质量 | 单选 | 好/一般/差 |

**AI 解析示例：**
```
用户："昨晚 11 点睡，今早 7 点起，睡得不错"
    ↓
AI 解析：
{
  "sleep_time": "23:00",
  "wake_time": "07:00",
  "quality": "好"
}
```

#### 2.3.2 运动记录

**记录方式：**
- AI 对话式："今天跑步 30 分钟"
- 手动录入：点击"记录运动"按钮

**字段：**

| 字段 | 类型 | 说明 |
|------|------|------|
| 日期 | 日期 | 默认今天 |
| 运动类型 | 单选 | 跑步/走路/游泳/力量训练/瑜伽/其他 |
| 时长 | 数字 | 单位：分钟 |
| 备注 | 文本 | 可选 |

**AI 解析示例：**
```
用户："今天跑步 30 分钟"
    ↓
AI 解析：
{
  "type": "跑步",
  "duration": 30
}
```

#### 2.3.3 饮水记录

**记录方式：**
- AI 对话式："喝了 500ml 水"

**字段：**

| 字段 | 类型 | 说明 |
|------|------|------|
| 日期 | 日期 | 默认今天 |
| 饮水量 | 数字 | 单位 ml |

**快捷操作：**
- 页面上显示 +250ml / +500ml 按钮

#### 2.3.4 排便记录

**记录方式：**

| 字段 | 类型 | 说明 |

## 3. 页面设计

### 3.1 身体数据主页（/data）

**页面结构：**
```
┌─────────────────────────────┐
│ 顶部：趋势图（默认体重）     │
│ [7天] [30天] [90天] [365天] │
├─────────────────────────────┤
│ Tab 切换：                  │
│ [体重] 围度 睡眠 运动 饮水 排便 │
├─────────────────────────────┤
│ 今日记录卡片（pending/recorded）│
├─────────────────────────────┤
│ 历史记录列表                │
├─────────────────────────────┤
│ [📷] [🎤] 说点什么...  [发送] │
└─────────────────────────────┘
```

**Tab 说明：**
- 体重：体重趋势图 + 今日体重卡片 + 历史记录
- 围度：围度趋势图 + 今日围度卡片 + 历史记录
- 睡眠：睡眠时长趋势 + 今日睡眠卡片 + 历史记录
- 运动：运动时长趋势 + 今日运动卡片 + 历史记录
- 饮水：饮水量趋势 + 今日饮水卡片 + 快捷按钮
- 排便：排便记录列表

### 3.2 卡片状态

| 状态 | 说明 | 显示 |
|------|------|------|
| empty | 未记录 | "+ 点击记录" 提示 |
| pending | 待确认 | 显示内容 + 操作按钮 |
| recorded | 已记录 | 显示内容，无操作按钮 |

### 3.3 趋势图设计

**时间范围切换：**
- 7 天：显示每日数据点
- 30 天：显示每日数据点
- 90 天：显示每日数据点（可能较密集）
- 365 天：显示每周平均值（避免过于密集）

**图表元素：**
- 折线图：主数据线
- 目标线：虚线标注目标值（如目标体重）
- 数据点：可点击查看详情
- 统计信息：最高/最低/平均值

## 4. 跨页面操作处理

### 4.1 在其他页面记录身体数据

**场景：** 用户在饮食页面说"今天体重 74.5kg"

**处理方式：**
```
AI 识别为 record_weight
    ↓
Toast 提示："正在记录体重..."
    ↓
自动跳转到数据页面（体重 Tab）
    ↓
体重卡片更新为 pending 状态
    ↓
显示 [✓ 确认] [✏️ 修改] [✗ 取消]
```

### 4.2 操作规则

| 当前页面 | AI 识别意图 | 处理方式 |
|---------|------------|---------|
| 数据页面 | record_weight | 直接更新体重卡片 |
| 其他页面 | record_weight | Toast + 跳转到数据页面 + 更新卡片 |
| 数据页面 | modify_weight | 直接更新体重卡片 |
| 其他页面 | modify_weight | Toast + 跳转到数据页面 + 更新卡片 |

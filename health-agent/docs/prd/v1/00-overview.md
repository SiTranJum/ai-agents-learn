# V1 内测版 - 总览

> 版本：V1 内测版
> 日期：2026-04-27
> 状态：设计中
> 目标：功能完整的内测版，让内部用户体验完整产品形态

---

## 1. V1 版本目标

### 1.1 核心目标
- 验证完整产品闭环能否成立
- 验证内部用户能否连续使用
- 验证 AI 是否真正提升体验
- 验证核心模块组合是否顺畅

### 1.2 用户范围
- 内部测试用户（5-10 人）
- 对产品有一定容忍度，能接受轻量化实现
- 愿意提供反馈和建议

### 1.3 版本原则
- **非 AI 核心功能尽量首版上线**：页面、表单、数据展示、基础交互
- **AI 核心能力必须全部具备**：但允许轻量化实现
- **不追求极致性能和完美容错**：但要保证核心链路可用
- **功能完整优先于体验完美**：先跑通，再优化

### 第三批（并行，预计 2-2.5 周）
- **03 饮食记录**：文本记录、拍照识别、营养查询、记录管理

### 第四批（并行，预计 1.5-2 周）

**总计：6-8 周**

## 5. V1 轻量化标准

对于复杂的 AI 能力，V1 不追求完美，但要能用：

| 模块 | V1 标准 | 不追求 |
|------|---------|--------|
| 拍照识别 | 识别常见食物，70-80% 准确率，允许用户纠正 | 极高精度、复杂混合菜品细分 |
| 三层记忆 | 基础架构搭建，4 个提取触发点，优先级召回 | 极致性能、复杂记忆治理 |
| RAG 知识库 | 覆盖 500-1000 种常见食物和基础健康知识 | 完整食物百科、医学级专业知识 |
| 计划修改 | 基础修改能力，规则检测 + LLM 建议 | 智能自动重规划、复杂阶段策略 |
| 交互模式 | 三种模式基础实现，影响核心场景 | 所有细节 100% 适配 |
| AI 建议 | 不胡说、有针对性、能体现记忆 | 营养师级专业建议、长篇科普 |

## 6. V1 交付标准

### 功能完整性
- ✅ 用户能注册、登录、配置档案
- ✅ 用户能通过文本和拍照记录饮食
- ✅ 用户能记录体重、围度等身体数据
- ✅ 用户能创建和执行健康计划
- ✅ 用户能查看数据分析和 AI 建议
- ✅ AI 能记住用户的偏好和行为

### 质量标准
- 饮食记录成功率 > 85%
- 拍照识别成功率 > 70%
- AI 建议不出现明显错误
- 用户能感知到"AI 记得我"
- 核心链路无阻塞性 bug

### 用户体验标准
- 内部测试用户能连续使用 7 天以上
- 用户反馈产品"能用"且"有价值"
- 用户愿意继续使用并提供反馈

## 9. 技术栈确认

| 技术领域 | 技术选型 | 说明 |
|---------|---------|------|
| 移动端框架 | React Native | 跨平台开发，JS/TS 生态 |
| 后端框架 | Python + FastAPI | AI 生态最成熟 |
| 数据库 | Supabase (PostgreSQL) | 关系型数据 + Auth |
| 向量数据库 | Supabase pgvector | PostgreSQL 向量插件，统一数据库 |
| LLM | 通义千问 (qwen-plus) | 学习阶段使用国内 LLM |
| Embedding | text-embedding-v3 | 通义千问 Embedding 模型 |
| AI 框架 | LangGraph | Agent 编排 |
| 图片识别 | 多模态 LLM | 通义千问 VL 或其他 |

**注：** V1 仅支持移动端（iOS/Android），不支持桌面端和平板端。

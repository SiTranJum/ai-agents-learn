# V1 - AI 建议系统

> 模块：AI 建议系统
> 优先级：P1（第四批开发）
> 依赖：03-饮食记录、05-AI 记忆系统、06-RAG 知识库
> 预计工时：1-1.5 周

---

## 1. 模块目标

基于用户的饮食记录、身体数据、健康目标和记忆，生成有针对性的健康建议。

### 1.1 核心目标
- 建议不胡说（基于 RAG 知识库）
- 建议有针对性（基于用户数据）
- 建议能体现记忆（基于用户偏好）
- 建议围绕目标（基于用户计划）

### 1.2 V1 标准
- 建议准确、不出现明显错误
- 建议个性化，不是通用模板
- 不追求营养师级专业建议

## 2. 建议类型

### 2.1 V1 支持的建议类型

| 建议类型 | 触发时机 | 内容 | 展示位置 |
|---------|---------|------|---------|
| 饮食建议 | 记录饮食后 | 基于当餐/当日营养分析 | Command Bar 结果卡片 |
| 目标建议 | 查看计划时 | 基于计划进度和执行情况 | 计划详情页 |
| 趋势建议 | 查看数据分析时 | 基于近期数据趋势 | 数据分析页 |
| 主动洞察 | 首页加载时 | 基于今日数据的一句话洞察 | 首页 Dashboard |

### 2.2 各类型建议详细说明

#### 饮食建议
**触发：** 用户记录饮食后

**分析维度：**
- 当餐热量是否合理
- 当日累计热量 vs 目标
- 营养素比例是否均衡
- 是否缺少某类营养素

**建议示例：**
- "午餐碳水偏高（65%），建议晚餐减少主食，增加蔬菜和蛋白质"
- "今天蛋白质摄入只有 30g，离目标 90g 还差很多，晚餐可以吃鸡胸肉或鱼"
- "今天热量已经达到 1600 kcal，离目标 1800 还有 200 kcal 的空间"

#### 目标建议
**触发：** 用户查看计划详情时

**分析维度：**
- 计划执行天数和达标率
- 近期趋势（在改善还是恶化）
- 与目标的差距

**建议示例：**
- "减脂计划执行 2 周，体重从 75kg 降到 74.2kg，进展顺利"
- "最近 3 天热量都超标 20%，建议关注午餐的主食份量"
- "本周达标率只有 40%，目标可能设得偏高，要不要调整一下？"

#### 趋势建议
**触发：** 用户查看数据分析页时

**分析维度：**
- 热量趋势（近 7 天/30 天）
- 营养素趋势
- 体重趋势
- 饮食规律性

**建议示例：**
- "近一周平均热量 1950 kcal，比上周增加了 150 kcal，主要是晚餐热量偏高"
- "你的蛋白质摄入在持续改善，从上周的 50g/天提升到了 70g/天"
- "体重近两周没有变化，可能进入了平台期，建议调整饮食结构"

#### 主动洞察
**触发：** 首页加载时

**内容：** 一句话总结今日/近期状态

**建议示例：**
- "今天还没记录饮食，别忘了记录哦"
- "今天蛋白质摄入偏低，晚餐记得补充"
- "连续 5 天达标，继续保持！"
- "今天是减脂计划第 14 天，已减重 0.8kg"

## 3. 建议生成逻辑

### 3.1 生成流程

```
触发建议生成
    ↓
[1] 收集用户数据
    - 今日饮食记录
    - 近期数据趋势
    - 用户档案和目标
    - 当前计划
    ↓
[2] 召回记忆
    - 长期记忆：偏好、限制
    - 中期记忆：近期行为模式
    ↓
[3] 检索知识库
    - 相关营养知识
    - 相关健康建议
    ↓
[4] LLM 生成建议
    - 基于数据 + 记忆 + 知识
    - 根据交互模式调整输出风格
    ↓
[5] 输出建议
```

### 3.2 交互模式对建议的影响

| 模式 | 建议风格 | 示例 |
|------|---------|------|
| 效率模式 | 一句话，直接给结论 | "晚餐建议吃鸡胸肉 + 蔬菜" |
| 确认模式 | 2-3 句，结论 + 简要原因 | "今天蛋白质偏低，建议晚餐吃鸡胸肉或鱼肉补充，目标还差 40g" |
| 学习模式 | 详细讲解，结论 + 原因 + 知识 | "今天蛋白质只摄入了 50g，目标是 90g。蛋白质对减脂很重要，因为..." |

### 3.3 建议质量规则

**必须遵守：**
- 不给出医疗诊断或治疗建议
- 不推荐用户过敏或忌口的食物
- 不建议极端饮食（如每日热量 < 基础代谢）
- 建议基于用户实际数据，不是泛泛而谈

**尽量做到：**
- 建议具体可执行（"吃鸡胸肉"而不是"多吃蛋白质"）
- 考虑用户偏好（不推荐用户不喜欢的食物）
- 语气友好，不说教

## 4. 建议展示

### 4.1 展示位置

| 位置 | 建议类型 | 展示方式 |
|------|---------|---------|
| 首页 Dashboard | 主动洞察 | 卡片，一句话 |
| Command Bar 结果 | 饮食建议 | 结果卡片底部附带 |
| 计划详情页 | 目标建议 | 卡片 |
| 数据分析页 | 趋势建议 | 卡片 |

### 4.2 建议卡片设计

```
┌─────────────────────────────────┐
│ 💡 AI 建议                      │
│                                 │
│ 今天蛋白质摄入偏低（50g/90g），  │
│ 建议晚餐吃鸡胸肉或鱼肉补充。    │
│                                 │
│ [有用 👍]  [没用 👎]  [关闭]    │
└─────────────────────────────────┘
```

### 4.3 建议反馈
- 用户可以对建议点"有用"或"没用"
- 反馈数据作为记忆提取的触发点
- 用于优化后续建议质量

## 5. 业务规则

### 5.1 建议频率
- 饮食建议：每次记录饮食后生成（确认模式和学习模式）
- 效率模式下不主动展示饮食建议，除非有明显问题
- 主动洞察：首页每次加载生成一条
- 目标建议和趋势建议：用户查看对应页面时生成

### 5.2 建议去重
- 同一天不重复给出相同建议
- 如果用户已经改善了某个问题，不再提醒

### 5.3 免责声明
- 所有建议附带免责声明："以上建议仅供参考，不构成医疗建议"
- 涉及疾病相关建议时，提醒用户咨询医生

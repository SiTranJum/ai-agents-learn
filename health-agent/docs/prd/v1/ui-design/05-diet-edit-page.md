# 饮食编辑详情页

## 1. 页面目标
提供表单化的饮食记录编辑能力，让用户可以精确修改食物名称、份量和单位，系统自动计算热量和营养素。

本页面是 AI 自然语言记录的补充兜底：
- 当 AI 解析结果不够准确时，用户可以在这里手动修正。
- 当用户需要精确控制份量（如称重记录）时，表单比对话更高效。
- 支持新增记录和编辑已有记录两种场景。

## 2. 页面入口
- 饮食记录页点击餐次卡片中的 [✏️ 修改] 按钮。
- 饮食记录页点击已记录卡片展开后的"编辑"入口。
- 待确认卡片点击"修改"按钮。

## 3. 页面布局

### 3.1 整体布局说明
页面从上到下依次为：
1. 顶部导航栏：返回按钮 + 页面标题
2. 餐次选择器：选择当前编辑的餐次
3. 食物列表：每个食物一行，支持增删改
4. 添加食物按钮
5. 营养汇总区：实时计算的热量和营养素
6. 底部操作栏：保存按钮 + 取消按钮

### 3.3 布局细节说明
- 顶部导航栏固定，标题根据餐次动态变化（如"编辑午餐记录"、"新增早餐记录"）。
- 餐次选择器为横向排列的标签组，当前选中项高亮。
- 食物列表区域可滚动，每个食物条目为一个独立卡片。
- 每个食物卡片包含：食物名称输入框、份量输入框、单位选择器、热量显示、删除按钮。
- 添加食物按钮位于列表末尾，点击后在列表底部新增一个空白食物条目。
- 营养汇总区实时计算所有食物的总热量和三大营养素。
- 底部操作栏固定在页面底部，保存和取消按钮并排。

### 4.1 重点组件说明
- FoodNameInput：文本输入框，支持模糊搜索食物库，输入时弹出候选列表。
- PortionInput：数字输入框，支持小数（如 0.5）。
- UnitSelector：下拉选择器，常见单位包括：克(g)、碗、个、片、杯、勺、份。
- CalorieDisplay：只读显示，灰色文字，根据食物名称 + 份量 + 单位自动计算。
- NutritionSummary：实时汇总区，随食物列表变化自动更新。

### 5.4 表单校验失败
适用场景：用户点击保存但存在必填项未填或数据异常。

#### 校验规则
- 至少有一个食物条目。
- 每个食物条目的名称不能为空。
- 份量必须大于 0。
- 餐次必须已选择。

#### 错误提示方式
- 对应输入框下方显示红色错误提示文字。
- 保存按钮不可点击（或点击后滚动到第一个错误位置）。

## 6. 关键交互

### 6.1 修改食物名称 / 份量 → 热量自动重算
- 用户修改食物名称后，系统查询食物库匹配营养数据。
- 用户修改份量或单位后，系统根据新的份量重新计算热量。
- 热量字段实时更新，无需手动触发。
- 底部营养汇总区同步刷新。
- 如果食物名称在食物库中未找到，热量显示"--"并提示"未找到该食物，请手动输入热量"。

### 6.2 删除食物 → 从列表移除
- 点击食物条目右下角的删除按钮。
- 该食物条目从列表中移除。
- 营养汇总区重新计算。
- 如果删除后列表为空，自动添加一个空白条目。
- 删除操作无需二次确认（页面级取消可恢复）。

### 6.3 添加食物 → 新增一行
- 点击"添加食物"按钮。
- 在列表末尾新增一个空白食物条目。
- 光标自动聚焦到新条目的食物名称输入框。
- 页面自动滚动到新条目位置。

### 6.4 保存 → 返回饮食记录页，卡片更新
- 点击保存按钮。
- 系统执行表单校验。
- 校验通过后保存数据。
- 返回饮食记录页。
- 对应餐次卡片更新为最新内容。
- 汇总区数据同步刷新。
- 提示"已保存午餐记录"。

### 6.5 取消 → 返回，不保存
- 点击取消按钮或顶部返回按钮。
- 如果表单有未保存的修改，弹出确认对话框："当前修改尚未保存，确认离开吗？"
- 用户确认后返回饮食记录页，不保存任何变更。
- 如果表单无修改，直接返回。

### 6.6 食物名称搜索与自动填充
- 用户在食物名称输入框中输入 2 个字符以上时，触发食物库模糊搜索。
- 搜索结果以下拉列表形式展示，最多显示 5 条候选。
- 点击候选项后：
  - 自动填充食物名称
  - 自动填充默认份量和单位
  - 自动计算热量
- 用户也可以忽略候选列表，手动输入自定义食物名称。

# 首页 Dashboard

> 页面编号：P01
> 对应 Tab：首页（底部导航第 1 个）
> 设计系统引用：`01-design-system.md`

---

## 1. 页面目标

首页用于展示今日健康概览、快捷记录入口、AI 洞察和计划进度，让用户打开 App 就能一眼看到今天的健康全貌，并快速完成记录操作。

---

## 2. 页面入口

底部 Tab 导航的"首页"Tab（图标：房子线性图标，选中态为珊瑚橙 `#FF7A5C` 填充）。App 启动后默认进入此页面。

## 5. 页面状态

### 5.1 新用户空态

用户刚注册完成、尚无任何记录数据时的状态。

| 区域 | 展示差异 |
|------|---------|
| 今日概览大卡片 | 热量环形图显示 0/目标值，三大营养素均为 0g，环形图为空灰色轨道 |
| 快捷操作区 | 正常显示，"记录饮食"按钮可加轻微呼吸动画引导点击 |
| 今日饮食时间轴 | 四餐均显示"未记录"，文案为"告诉 AI 你吃了什么" |
| AI 洞察卡片 | 显示欢迎语："欢迎使用健康管家！试试在下方输入框记录你的第一餐吧。" |
| 计划进度卡片 | 显示空态插画 + 文案"还没有计划，和 AI 聊聊你的健康目标吧"，点击可跳转创建 |
| 辅助记录小卡片 | 各项显示"--"或"未记录"，图标保持但数值为空 |
| AI 输入框 | placeholder 文案："试试说「我早餐吃了全麦面包和牛奶」" |

### 5.2 有数据态

用户已有部分或全部记录数据时的正常状态。

| 区域 | 展示差异 |
|------|---------|
| 今日概览大卡片 | 环形图按实际热量填充，颜色根据达标情况变化（正常绿、接近黄、超标红） |
| 今日饮食时间轴 | 已记录餐次显示食物摘要 + 热量，未记录餐次显示引导文案 |
| AI 洞察卡片 | 根据当日数据生成个性化建议 |
| 计划进度卡片 | 显示当前进行中计划的进度和今日任务完成情况 |
| 辅助记录小卡片 | 显示实际数值，饮水显示进度（如 1200/2000ml） |

### 5.3 局部待确认态

用户通过 AI 输入框记录了饮食，AI 解析完成但尚未确认时的状态。

| 区域 | 展示差异 |
|------|---------|
| 今日饮食时间轴 | 对应餐次卡片高亮边框（`brand-primary-light` #FFE7E1 背景），显示 AI 解析出的食物内容，底部出现 [确认] [修改] [取消] 三个操作按钮 |
| 今日概览大卡片 | 热量和营养素数值暂不更新，保持确认前的数据，避免误导 |
| 页面滚动 | 自动滚动到待确认的餐次卡片位置，确保用户能看到 |

## 6. 关键交互

### 6.1 点击快捷操作

| 操作 | 行为 |
|------|------|
| 点击"记录饮食" | 跳转到饮食记录页，自动定位到当前时段对应的餐次（上午→早餐，中午→午餐，下午→加餐，晚上→晚餐） |
| 点击"记录体重" | 弹出体重输入底部面板（数字键盘 + 体重输入框），输入后保存并更新数据页 |
| 点击"查看计划" | 跳转到计划列表页；若无计划，跳转到计划创建全屏对话页 |

### 6.2 点击今日饮食卡片

- 点击整个"今日饮食"卡片标题区域的"查看全部"：跳转到饮食记录页，展示今日全部饮食详情。
- 点击某一餐次（如"午餐"行）：跳转到饮食记录页并定位到该餐次。
- 点击未记录餐次：跳转到饮食记录页对应餐次，输入框自动聚焦。

### 6.3 点击计划卡片

- 点击"查看计划"链接或卡片任意区域：跳转到计划详情页，展示当前进行中计划的完整信息。
- 若有多个计划，跳转到计划列表页。

### 6.4 首页通过 AI 输入框记录饮食/体重

**记录饮食流程：**
1. 用户在底部输入框输入"午餐吃了一碗米饭和番茄炒蛋"
2. 输入框显示发送中状态（发送按钮变为 loading）
3. AI 解析完成后，今日饮食时间轴中"午餐"行切换为待确认态（高亮背景 + 解析内容 + 操作按钮）
4. 页面自动滚动到待确认卡片位置
5. 用户点击 [确认]：卡片切换为已记录态，今日概览大卡片的热量和营养素数值同步更新
6. 用户点击 [修改]：跳转到饮食记录编辑详情页
7. 用户点击 [取消]：卡片恢复为未记录态

**记录体重流程：**
1. 用户输入"今天体重 65.5 公斤"
2. AI 解析后弹出确认 Toast："已记录体重 65.5kg，查看详情"
3. 辅助记录区域不直接显示体重（体重在数据页），但 Toast 提供跳转入口

**跨页面操作：**
- 若用户输入"帮我制定一个减脂计划"，弹出跨页面确认弹窗，引导跳转到计划创建页面

### 6.5 辅助记录小卡片交互

| 操作 | 行为 |
|------|------|
| 点击饮水卡片 | 弹出底部面板，显示快捷按钮 +250ml / +500ml / 自定义 |
| 点击睡眠卡片 | 跳转到数据页睡眠 Tab |
| 点击运动卡片 | 跳转到数据页运动 Tab |
| 点击排便卡片 | 弹出底部面板，选择状态（正常/便秘/腹泻）+ 时间 |

### 6.6 AI 洞察卡片交互

- 点击"查看详情"：跳转到 AI 建议详情页，展示完整建议内容、原因和可执行动作。

## 8. Figma AI 生成描述

设计一个移动端 App 的首页 Dashboard 页面，健康生活方式风格，整体温暖亲切，不要过度科技感。页面采用卡片式布局，白色卡片搭配浅灰背景（#F7F8FA），卡片圆角 12px，带轻柔阴影。主色为珊瑚橙（#FF7A5C），用于按钮、进度条和强调元素。

页面从上到下依次排列：顶部标题区显示"今天"和当前日期；紧接着是一张大卡片展示今日热量概览，左侧是珊瑚橙色的环形进度图显示已摄入热量和目标热量（如 1842/2100 kcal），中间纵向排列蛋白质、脂肪、碳水三项营养素及其克数和小进度条，右侧放一个扁平风格的健康人物插画（年轻女性拿着沙拉碗，颜色柔和线条圆润）；下方是三个等宽的快捷操作胶囊按钮（记录饮食、记录体重、查看计划），每个按钮有小图标和文字；再下方是今日饮食时间轴卡片，左侧用竖线连接四个餐次节点（早餐、午餐、晚餐、加餐），每个节点有对应的小图标，已记录的餐次显示食物摘要和热量，未记录的显示浅灰色引导文字；接着是 AI 洞察卡片，带灯泡图标，显示一句个性化饮食建议；然后是计划进度卡片，显示计划名称、珊瑚橙进度条和今日任务完成数；最后是 2x2 网格的辅助记录小卡片（饮水、睡眠、运动、排便），每个小卡片有对应的线性图标和数值。

页面底部固定两层：第一层是 AI 输入框（白色背景，胶囊型输入框，左侧有拍照和语音按钮，右侧有发送按钮）；第二层是底部 Tab 导航（首页/饮食/数据/我的，首页 Tab 高亮为珊瑚橙）。整体信息丰富但层级清晰，大卡片和小卡片混排，间距舒适不拥挤。适配 iPhone 14/15 尺寸，中文界面。参考健康类 App 如薄荷健康、MyFitnessPal 的首页布局风格，但更温暖、更生活化。

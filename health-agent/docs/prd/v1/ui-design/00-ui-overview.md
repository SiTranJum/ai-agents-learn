# V1 UI 设计总览

> 目录：UI 设计文档
> 用途：提供给 AI 生成页面、组件和素材的完整描述
> 适用范围：健康管家 AI Agent V1 内测版（移动端 App）

---

## 1. 设计目标

为 V1 的所有页面提供统一的视觉和交互规范，让 AI 能根据文档直接生成：
- 页面 UI
- 组件样式
- 图标和插画素材
- 交互状态

**设计原则：**
- 温和、亲切、健康感
- AI-first，但不过度科技感
- 卡片式布局，信息清晰
- 移动端单手操作友好
- 输入优先，鼓励用户依赖 AI 交互

---

## 2. 全局视觉规范

### 2.1 整体风格
- 风格：轻健康、生活化、亲和力强
- 避免：重医疗感、重科技感、深色压迫感
- 感受关键词：温暖、柔和、清爽、可信赖

### 2.2 配色方案

**主色：**
- 主品牌色：珊瑚橙 `#FF7A5C`
- 主品牌深色：`#E96345`
- 主品牌浅色：`#FFE7E1`

**辅助色：**
- 成功绿：`#4CD964`
- 警告黄：`#FFC94D`
- 信息蓝：`#5AC8FA`
- 错误红：`#FF5A5F`

**中性色：**
- 页面背景：`#F7F8FA`
- 卡片背景：`#FFFFFF`
- 一级文字：`#222222`
- 二级文字：`#666666`
- 三级文字：`#999999`
- 分割线：`#EEEEEE`

### 2.3 字体规范
- 中文字体：系统默认无衬线字体
- 标题：18-24px，600-700 粗细
- 正文：14-16px，400-500 粗细
- 辅助说明：12-13px，400 粗细
- 数值重点：28-40px，700 粗细

### 2.4 圆角与阴影
- 小按钮圆角：8px
- 卡片圆角：12px
- 重要弹层圆角：16px
- 输入框圆角：24px（胶囊型）
- 阴影：轻阴影，柔和，不要厚重

### 2.5 插画风格
- 扁平插画 + 轻微拟人风
- 颜色柔和，线条圆润
- 场景以健康、饮食、运动、习惯养成为主
- 人物插画以年轻女性和中性风为主，友好不夸张

## 3. 全局布局规范

### 3.1 页面结构
所有核心业务页面统一采用以下结构：

```
┌─────────────────────────────┐
│ 顶部标题区 / 页面头部        │
├─────────────────────────────┤
│                             │
│        页面内容区            │
│                             │
├─────────────────────────────┤
│ [📷] [🎤] 说点什么... [发送] │ ← AI 输入框（常驻）
├─────────────────────────────┤
│  🏠    🍽️    📊    👤       │ ← 底部 Tab 导航
└─────────────────────────────┘
```

### 3.2 底部双层固定区

**第一层：AI 输入框**
- 高度约 56px
- 白色背景，轻阴影
- 左侧拍照按钮、语音按钮
- 中间输入框
- 右侧发送按钮
- 永久常驻

**第二层：底部 Tab 导航**
- 高度约 60px
- 4 个 Tab：首页 / 饮食 / 数据 / 我的
- 当前 Tab 高亮为珊瑚橙
- 图标线性、简洁、圆润

## 5. 页面详细设计描述

### 5.1 登录页

**页面目标：** 用户输入邮箱密码登录

**布局：**
- 顶部留白较大，展示 Logo 和品牌名
- 中间是登录表单卡片
- 底部是注册入口和忘记密码入口

**内容：**
- Logo：简洁健康风图标（心形 + 叶子 / 餐盘 + 对话气泡）
- 标题："欢迎回来"
- 副标题："像朋友一样，陪你管理健康"
- 输入框 1：邮箱
- 输入框 2：密码
- 主要按钮："登录"
- 次要文字按钮："忘记密码"
- 底部文字："还没有账号？去注册"

**风格：**
- 极简、温和
- 按钮使用主品牌色珊瑚橙
- 表单卡片白底圆角，带轻阴影

---

### 5.2 注册页

**页面目标：** 用户使用邮箱 + 密码注册

**内容：**
- 标题："创建你的健康档案"
- 输入框：邮箱、密码、确认密码
- 提示文案："V1 暂仅支持邮箱注册"
- 主按钮："注册"
- 底部文字："已有账号？去登录"

**注意：**
- 不展示第三方登录按钮
- 可以在页面底部小字提示："后续将支持 QQ 登录"

---

### 5.3 基础档案填写页

**页面目标：** 完成基础健康档案填写

**布局：**
- 分步式表单
- 顶部进度条
- 每一步 4-5 个字段
- 底部按钮："下一步" / "上一步"

**步骤建议：**
1. 基础信息：昵称、性别、出生日期
2. 身体信息：身高、当前体重、目标体重、活动量
3. 健康目标：减脂/增肌/保持/改善饮食/控制慢性病
4. 饮食偏好：素食、过敏、忌口、不喜欢食物
5. 疾病信息：高血压、糖尿病、痛风等

**视觉：**
- 每个步骤顶部有简洁图标
- 表单项间距大，适合移动端点击
- 可跳过非必填项

---

### 5.4 首页 Dashboard

**参考风格：** 你发的截图风格，卡片丰富、生活化

**页面内容：**
1. 顶部标题："首页" 或 "今天"
2. 今日概览大卡片：
   - 今日热量摄入 / 目标热量
   - 蛋白质、脂肪、碳水概览
   - 右侧可有轻插画（健康人物/饮食插画）
3. 快捷操作卡片：
   - 记录饮食
   - 记录体重
   - 查看计划
4. 今日饮食卡片：
   - 早餐 / 午餐 / 晚餐 / 加餐
   - 每餐展示已记录内容或"未记录"
5. AI 洞察卡片：
   - 一句建议
   - 可展开详情
6. 计划进度卡片：
   - 当前计划名称
   - 完成度进度条
   - 今日任务数
7. 辅助记录小卡片区：
   - 饮水
   - 睡眠
   - 运动
   - 排便
   - 健康习惯

**风格重点：**
- 卡片多，但层级清晰
- 大卡片 + 小卡片混排
- 每个卡片都像独立信息模块

---

### 5.5 饮食记录页

**页面目标：** 按餐次展示饮食卡片，并支持 AI 直接更新卡片

**布局：**
- 顶部：日期切换 + 今日汇总
- 中部：早餐 / 午餐 / 晚餐 / 加餐卡片列表
- 底部：AI 输入框常驻

**卡片状态设计：**

#### 状态 1：未记录
- 卡片标题：早餐 / 午餐 / 晚餐 / 加餐
- 中间显示："+ 点击记录" 或 "告诉 AI 你吃了什么"
- 浅灰背景，弱提示

#### 状态 2：待确认
- 卡片高亮边框（珊瑚橙浅色）
- 显示 AI 刚解析出的食物内容
- 卡片底部有按钮：
  - [✓ 确认]
  - [✏️ 修改]
  - [✗ 取消]

#### 状态 3：已记录
- 正常白色卡片
- 展示食物列表、热量、营养素
- 可点击进入详情

**页面顶部汇总：**
- 今日总热量 / 目标热量
- 环形图：碳水 / 蛋白质 / 脂肪占比
- 简单进度条

### 5.6 饮食记录编辑详情页

**页面目标：** 用户点击修改后进入详情页，用表单编辑当前记录

**风格：**
- 表单卡片式布局

### 5.7 数据页

**页面目标：** 展示体重、围度、睡眠、运动、饮水、排便等数据

**布局：**

- 中部：Tab 切换（体重 / 围度 / 睡眠 / 运动 / 饮水 / 排便）

**体重卡片：**

- 副信息：比昨天变化

**围度卡片：**

**饮水卡片：**
- 水杯插画
- 今日已喝 / 目标
- 快捷按钮：+250ml / +500ml

**睡眠卡片：**
- 昨晚睡眠时长

- 睡眠质量标签

## 7. 给 AI 生成页面的统一提示模板

当用 AI 生成页面时，统一使用以下描述：

- 平台：移动端 App（React Native 风格）
- 风格：轻健康、温和、卡片式布局
- 主色：珊瑚橙 #FF7A5C
- 背景：浅灰白 #F7F8FA
- 卡片：白底、12px 圆角、轻阴影
- 输入框：底部常驻胶囊型输入框
- 导航：底部 4 Tab
- 适配：iPhone 14/15 尺寸优先
- 语言：中文

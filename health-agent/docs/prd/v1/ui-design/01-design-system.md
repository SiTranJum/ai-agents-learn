# 设计系统规范

> 健康管家 AI Agent 移动端 App 设计系统。本文档定义所有视觉基础 Token，可直接用于 Figma AI 生成组件和页面。

---

## 1. 配色

### 1.1 品牌色

| Token 名称 | 色值 | 用途 |
|-----------|------|------|
| `brand-primary` | `#FF7A5C` 珊瑚橙 | 主按钮、Tab 高亮、进度条、强调元素 |
| `brand-primary-dark` | `#E96345` | 按钮按下态、深色文字强调 |
| `brand-primary-light` | `#FFE7E1` | 待确认卡片背景、浅色标签、选中态背景 |

**品牌色使用规则：**
- 每个页面最多 1 个 `brand-primary` 主按钮，避免视觉竞争
- `brand-primary-light` 用于需要柔和强调的背景区域，如 AI 待确认的饮食记录卡片
- 进度条填充使用 `brand-primary`，轨道背景使用 `#F0F0F0`

### 1.2 功能色

| Token 名称 | 色值 | 用途 | 使用场景举例 |
|-----------|------|------|------------|
| `color-success` | `#4CD964` | 达标、完成、正常状态 | 热量达标、目标完成打勾、BMI 正常 |
| `color-warning` | `#FFC94D` | 接近目标、注意提示 | 热量接近上限（80%-100%）、轻度超标 |
| `color-info` | `#5AC8FA` | 链接、信息提示 | 可点击链接、信息类 Toast、饮水提醒 |
| `color-error` | `#FF5A5F` | 超标、错误、删除 | 热量严重超标、输入错误、删除确认 |

**功能色使用规则：**
- 功能色仅用于传达状态语义，不用于装饰
- 功能色可搭配 10% 透明度作为背景色（如 `rgba(76,217,100,0.1)` 作为达标卡片背景）

### 1.3 中性色

| Token 名称 | 色值 | 用途 |
|-----------|------|------|
| `bg-page` | `#F7F8FA` | 页面背景 |
| `bg-card` | `#FFFFFF` | 卡片背景 |
| `text-primary` | `#222222` | 标题、重要数值 |
| `text-secondary` | `#666666` | 正文、说明 |
| `text-tertiary` | `#999999` | 辅助文字、placeholder |
| `border-divider` | `#EEEEEE` | 分割线 |
| `color-disabled` | `#CCCCCC` | 禁用态文字、禁用态边框 |

**中性色使用规则：**
- 页面背景统一使用 `bg-page`，卡片使用 `bg-card`，形成层级对比
- 文字层级严格遵循三级体系：标题用 `text-primary`，正文用 `text-secondary`，辅助用 `text-tertiary`
- 分割线使用 1px `border-divider`，不使用边框描边

## 2. 字体

### 2.1 字体选择

| 平台 | 中文字体 | 英文/数字字体 |
|------|---------|-------------|
| iOS | PingFang SC | SF Pro |
| Android | Noto Sans CJK SC | Roboto |

**特殊场景：**
- 核心数值（体重、热量、步数等大数字）使用 DIN Alternate（iOS）或 Roboto Mono（Android），突出数据感
- Figma 设计稿统一使用 PingFang SC + DIN Alternate

### 2.2 字体层级表

| Token 名称 | 字号 | 粗细 | 行高 | 字间距 | 用途 | 示例 |
|-----------|------|------|------|--------|------|------|
| `font-hero` | 36px | Bold (700) | 44px | -0.5px | 核心数值展示 | "1,842" kcal、"65.5" kg |
| `font-page-title` | 24px | Bold (700) | 32px | 0 | 页面顶部标题 | "今日概览"、"饮食记录" |
| `font-card-title` | 18px | SemiBold (600) | 26px | 0 | 卡片内标题 | "早餐"、"今日营养" |
| `font-body` | 16px | Regular (400) | 24px | 0.2px | 主要内容文字 | AI 对话内容、食物名称 |
| `font-body-sm` | 14px | Regular (400) | 20px | 0.1px | 次要内容 | 列表项、营养素明细 |
| `font-caption` | 13px | Regular (400) | 18px | 0.1px | 辅助说明 | 时间戳、来源说明 |
| `font-tag` | 12px | Medium (500) | 16px | 0.2px | 标签、badge | "已达标"、"待确认" |

**字体使用规则：**
- 同一卡片内最多使用 3 个字体层级
- 数值 + 单位组合：数值使用 `font-hero`，单位使用 `font-caption`，如 "1,842 `kcal`"
- AI 对话气泡内文字统一使用 `font-body`

## 3. 间距系统

基础单位：**4px**（所有间距值必须是 4 的倍数）

### 3.1 间距 Token

| Token 名称 | 值 | 用途 |
|-----------|-----|------|
| `space-xs` | 4px | 图标与文字间距、紧凑元素内间距 |
| `space-sm` | 8px | 标签内边距、紧凑列表项间距、区域标题与内容间距 |
| `space-md` | 12px | 卡片间距、列表项间距 |
| `space-lg` | 16px | 页面左右边距、卡片内边距、区域间距 |
| `space-xl` | 24px | 大区域间距、页面顶部留白 |
| `space-xxl` | 32px | 页面底部留白、大区域分隔 |

### 3.2 页面间距规范

```
┌─────────────────────────────────┐
│         状态栏 (系统)            │
├─────────────────────────────────┤
│  ← 16px →  页面标题   ← 16px → │  页面左右边距: 16px
│                                 │
│  ┌───────────────────────────┐  │
│  │  ← 16px 卡片内边距 16px → │  │  卡片内边距: 16px
│  │                           │  │
│  │  卡片内容                  │  │
│  │                           │  │
│  └───────────────────────────┘  │
│          ↕ 12px 卡片间距         │  卡片间距: 12px
│  ┌───────────────────────────┐  │
│  │  下一张卡片                │  │
│  └───────────────────────────┘  │
│                                 │
│         ↕ 32px 底部留白          │
├─────────────────────────────────┤
│  AI 输入框 (56px)               │  底部安全区: 116px
│  底部 Tab 导航 (60px)           │  (Tab 60px + 输入框 56px)
└─────────────────────────────────┘
```

**间距使用规则：**
- 区域标题（如"今日营养"）与下方内容间距：`space-sm`（8px）
- 同一卡片内元素间距：`space-sm`（8px）或 `space-md`（12px）
- 不同卡片之间间距：`space-md`（12px）
- 页面底部需预留 116px 安全区（底部 Tab 导航 60px + AI 输入框 56px）

## 4. 圆角

| Token 名称 | 圆角值 | 使用元素 | 说明 |
|-----------|--------|---------|------|
| `radius-sm` | 8px | 小按钮、标签 | 操作按钮、状态标签 |
| `radius-md` | 12px | 卡片 | 所有卡片统一 |
| `radius-lg` | 16px（顶部） | 弹层、底部面板 | 确认弹窗、浮层卡片，仅顶部圆角 |
| `radius-pill` | 24px | 输入框、胶囊按钮 | AI 输入框、搜索框、胶囊操作按钮 |
| `radius-full` | 999px / 50% | 头像、圆形按钮 | 用户头像、圆形图标按钮 |

**圆角使用规则：**
- 卡片统一使用 `radius-md`，避免一个页面出现多种卡片圆角
- 底部弹层仅顶部使用 `radius-lg`，底部贴边不留圆角
- AI 输入框和搜索框使用 `radius-pill`，营造友好、轻松的对话感

## 5. 阴影

| Token 名称 | 阴影值 | 使用元素 | 说明 |
|-----------|--------|---------|------|
| `shadow-card` | `0 2px 8px rgba(0,0,0,0.06)` | 卡片 | 轻阴影，柔和 |
| `shadow-input` | `0 -2px 8px rgba(0,0,0,0.04)` | AI 输入框 | 向上阴影，区分层级 |
| `shadow-modal` | `0 -4px 16px rgba(0,0,0,0.1)` | 弹层、浮层 | 较重阴影，突出层级 |
| `shadow-brand-button` | `0 4px 12px rgba(255,122,92,0.3)` | 主按钮悬浮态 | 品牌色阴影 |

**阴影使用规则：**
- 页面常规卡片默认使用 `shadow-card`
- 底部 AI 输入框固定悬浮时使用 `shadow-input`
- 弹层、底部操作面板使用 `shadow-modal`
- 阴影只用于强调层级，不叠加多重阴影

## 6. 图标规范

### 6.1 图标风格

- **风格**：线性图标（Outline），圆角端点（round cap, round join）
- **线宽**：1.5px
- **默认颜色**：`#999999`（`text-tertiary`）
- **激活/选中颜色**：`#FF7A5C`（`brand-primary`）
- **不使用填充图标**，保持轻量感
- **推荐图标库**：Phosphor Icons（Light 风格）或 Feather Icons

### 6.2 图标尺寸

| Token 名称 | 尺寸 | 点击热区 | 用途 |
|-----------|------|---------|------|
| `icon-nav` | 24×24px | 44×44px | 底部 Tab 导航 |
| `icon-list` | 20×20px | 36×36px | 列表项前缀图标 |
| `icon-inline` | 16×16px | — | 文字内嵌图标 |
| `icon-action` | 28×28px | 44×44px | AI 输入框内的拍照、语音按钮 |

### 6.3 Tab 导航图标

| Tab | 图标描述 | 默认态 | 选中态 |
|-----|---------|--------|--------|
| 首页 | 房子轮廓（Home） | `#999999` 线性 | `#FF7A5C` 线性 |
| 饮食 | 餐盘 + 刀叉轮廓（Fork Knife） | `#999999` 线性 | `#FF7A5C` 线性 |
| 数据 | 柱状图轮廓（Chart Bar） | `#999999` 线性 | `#FF7A5C` 线性 |
| 我的 | 人物轮廓（User） | `#999999` 线性 | `#FF7A5C` 线性 |

**图标使用规则：**
- 所有可点击图标的点击热区不小于 44×44px（iOS Human Interface Guidelines）
- Tab 导航图标下方文字使用 `font-tag`（12px），间距 `space-xs`（4px）
- 图标与文字水平排列时，间距使用 `space-xs`（4px）

## 7. 插画规范

### 7.1 插画风格

- **风格**：扁平插画 + 轻微拟人
- **色调**：与品牌色系一致，柔和暖色为主（珊瑚橙、浅粉、暖黄）
- **线条**：圆润，无尖角，线宽 2px
- **人物**：简洁、友好、中性风格，无明显性别/年龄特征
- **场景**：健康、饮食、运动、习惯养成
- **背景**：透明或浅色渐变，不使用复杂场景

### 7.2 插画使用场景

| 场景 | 尺寸 | 说明 | 示例内容 |
|------|------|------|---------|
| 首页概览装饰 | 120×120px | 健康人物插画 | 做瑜伽的人物、举哑铃的人物 |
| 空状态 | 200×200px | 未记录、无计划、无数据 | 空盘子 + 问号、空白日历 |
| 卡片装饰 | 60×60px | 小型装饰插画 | 饮水杯、运动人物、睡眠月亮 |
| 登录/注册 | 240×240px | 品牌形象插画 | 健康生活场景、人物 + 食物 |
| 引导页 | 280×280px | 功能介绍插画 | 对话场景、拍照识别、数据图表 |

**插画使用规则：**
- 空状态插画下方搭配一行说明文字（`font-body`）和一个操作按钮
- 卡片装饰插画放置在卡片右侧或右上角，不遮挡主要内容
- 所有插画导出为 SVG 格式（矢量），支持多分辨率

## 8. 动效规范

### 8.1 基础动效

| 动效类型 | 时长 | 缓动函数 | 说明 |
|---------|------|---------|------|
| 页面切换 | 300ms | ease-in-out | 左右滑动过渡 |
| 卡片状态变化 | 200ms | ease | 背景色、边框色渐变过渡 |
| 弹窗出现 | 250ms | ease-out | 从底部滑入 + 背景蒙层渐显 |
| 弹窗消失 | 200ms | ease-in | 向下滑出 + 背景蒙层渐隐 |
| Toast 出现 | 200ms | ease-out | 从顶部滑入 |
| Toast 消失 | 300ms | ease-in | 向上滑出 + 透明度渐变 |
| 数值变化 | 400ms | ease-out | 数字滚动动画（如热量数值更新） |
| 进度条填充 | 500ms | ease-out | 从 0 到目标值的填充动画 |

### 8.2 交互反馈

| 交互类型 | 效果 | 时长 | 说明 |
|---------|------|------|------|
| 按钮点击 | 缩放至 0.95 | 100ms | 所有可点击按钮 |
| 卡片点击 | 背景色变浅至 `#F5F5F5` | 100ms | 可点击卡片的按下反馈 |
| 长按 | 缩放至 0.98 + 阴影加深 | 200ms | 长按删除、长按编辑 |
| 下拉刷新 | 旋转加载图标 | 持续 | 页面下拉刷新 |
| AI 思考中 | 三个点依次跳动 | 循环，每点 300ms | AI 回复等待状态 |

**动效使用规则：**
- 所有动效时长不超过 500ms，保持轻快节奏
- 用户操作反馈（点击、按下）控制在 100-200ms 内
- 尊重系统"减弱动态效果"设置，开启时关闭非必要动效
- 加载状态使用骨架屏（Skeleton），不使用传统转圈加载

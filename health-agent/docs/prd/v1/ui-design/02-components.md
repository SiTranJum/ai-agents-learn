# 全局组件规范

> 文档用途：定义健康管家 AI Agent 移动端 App 的全局组件标准，供产品、设计、前端开发与 AI 生成工具直接使用。
> 适用范围：iOS/Android 移动端 V1。
> 基础视觉：品牌色 `#FF7A5C`，页面背景 `#F7F8FA`，卡片白底 `#FFFFFF`，标准卡片圆角 `12px`。

---

## 0. 全局使用约定

### 0.1 统一视觉 Token

| 项目 | 规格 |
|------|------|
| 页面背景 | `#F7F8FA` |
| 卡片背景 | `#FFFFFF` |
| 主品牌色 | `#FF7A5C` |
| 主品牌浅色 | `#FFE7E1` |
| 一级文字 | `#222222` |
| 二级文字 | `#666666` |
| 三级文字 | `#999999` |
| 分割线 | `#EEEEEE` |
| 成功色 | `#4CD964` |
| 警告色 | `#FFC94D` |
| 信息色 | `#5AC8FA` |
| 错误色 | `#FF5A5F` |
| 卡片圆角 | `12px` |
| 浮层/弹窗圆角 | `16px` |
| 胶囊圆角 | `24px` |
| 标准阴影 | `0 4px 20px rgba(34,34,34,0.08)` |
| 轻阴影 | `0 -2px 12px rgba(34,34,34,0.06)` |

### 0.2 布局基准

- 基准画板宽度：`375px`
- 页面左右安全边距：`16px`
- 卡片之间垂直间距：`12px`
- 组件内部常规间距：`8px / 12px / 16px`
- 默认触控热区：不小于 `44x44px`
- 所有底部固定组件必须考虑 iPhone Home Indicator 安全区

### 0.3 输出给 AI 工具时的统一约束

- 组件风格：温和、轻健康、亲和，不要强科技感。
- 图标风格：线性圆角图标，线宽 1.5px~2px。
- 文案语气：像朋友提醒，简洁、自然，不要命令式。
- 除特别说明外，卡片默认白底、12px 圆角、无重描边、轻阴影或无阴影。
- 所有状态都必须保留同一组件骨架，避免状态切换时布局跳动。

## 1. 底部 Tab 导航

### 1.1 描述

底部 Tab 导航用于 App 一级信息架构切换，固定出现在主框架页面底部。包含 4 个 Tab：首页、饮食、数据、我的。整体风格简洁，弱化容器感，强调当前选中项。

### 1.2 尺寸规格

| 项目 | 规格 |
|------|------|
| 容器高度 | `60px`（不含底部安全区） |
| 安全区补偿 | `env(safe-area-inset-bottom)`，最小按 `16px` 预留 |
| 背景色 | `#FFFFFF` |
| 顶部分割线 | `1px #EEEEEE` |
| 单个 Tab 宽度 | `25%` 等分 |
| 图标尺寸 | `24x24px` |
| 图标与文字间距 | `4px` |
| 文字字号 | `12px`，Medium |
| 内容垂直布局 | 图标上、文字下，整体居中 |

### 1.3 状态

| 状态 | 图标颜色 | 文字颜色 | 说明 |
|------|----------|----------|------|
| 默认 | `#999999` | `#999999` | 非当前页面 |
| 选中 | `#FF7A5C` | `#FF7A5C` | 当前激活页面 |
| 按下反馈 | `#E96345` | `#E96345` | 手指按下瞬时反馈，100ms~150ms |
| 禁用 | `#CCCCCC` | `#CCCCCC` | 极少使用，仅预留 |

### 1.4 交互说明

- 点击任一 Tab，切换到对应一级页面。
- 当前页对应 Tab 保持选中态，不显示额外胶囊背景。
- 键盘弹起时，底部 Tab 导航可被输入区或系统键盘遮住，不强制悬浮显示。
- 适配 iPhone 底部安全区时，白色背景区域需要延伸到底部最下缘。

## 2. AI 输入框

### 2.1 描述

AI 输入框是全局核心交互入口，承担文字输入、拍照输入、语音输入和发送操作。通常固定在页面底部，强调“随时告诉 AI”的输入感。

### 2.2 尺寸规格

| 项目 | 规格 |
|------|------|
| 整体高度 | `56px` |
| 背景色 | `#FFFFFF` |
| 顶部阴影 | `0 -2px 12px rgba(34,34,34,0.06)` |
| 左右内边距 | `12px` |
| 上下内边距 | `8px` |
| 左侧按钮尺寸 | `28x28px` |
| 左侧两个按钮间距 | `8px` |
| 中间输入胶囊高度 | `40px` |
| 中间输入胶囊圆角 | `24px` |
| 输入胶囊背景 | `#F7F8FA` |
| Placeholder 颜色 | `#999999` |
| 发送按钮热区 | 最小 `44x44px` |
| 发送按钮图标 | `20x20px` 或文字按钮 |

### 2.3 布局结构

- 左侧：拍照按钮、语音按钮
- 中间：胶囊型输入框，宽度自适应填满剩余空间
- 右侧：发送按钮
- 整体组件固定底部，默认覆盖在页面内容上方

### 2.4 状态

| 状态 | 表现 |
|------|------|
| 默认 | 显示 placeholder `说点什么...`，发送按钮可弱化或不可点 |
| 输入中 | 键盘弹起，输入框获得焦点，组件整体上移到键盘上方，Tab 导航被遮住 |
| 录音中 | 语音按钮变红 `#FF5A5F`，按钮周围出现简洁波形动画或脉冲光圈 |
| 发送中 | 发送按钮替换为 loading，小型转圈动画，输入区可保持当前内容或锁定 |
| 禁用 | 网络不可用或处理中时，整体透明度降至 60%，按钮不可点 |

### 2.5 交互说明

- 拍照按钮：触发拍照或相册选择。
- 语音按钮：按下开始录音，松开结束；或点击切换录音模式，具体交互由页面决定，但视觉上统一使用红色录音态。
- 发送按钮：当输入为空时默认弱化；当有输入内容时使用品牌色高亮。
- 键盘弹起时，输入框随键盘上移，底部 Tab 导航允许被遮挡，这是全局规则。

### 2.6 ASCII 线框图

默认态：

│ [📷] [🎤]  ( 说点什么...         ) [发送] │

## 3. 餐次卡片

### 3.1 描述

餐次卡片用于饮食记录页，按早餐、午餐、晚餐、加餐等餐次展示。该组件承担“未记录 -> AI 解析待确认 -> 已记录”的完整状态流转，是饮食页最重要的业务卡片。

### 3.2 尺寸规格

| 项目 | 规格 |
|------|------|
| 卡片宽度 | 页面宽度减左右边距，自适应 |
| 最小高度 | `104px` |
| 背景 | `#FFFFFF` 或状态指定背景 |
| 圆角 | `12px` |
| 内边距 | `16px` |
| 标题字号 | `16px`，SemiBold |
| 内容字号 | `14px`~`16px` |
| 状态按钮高度 | `32px` |
| 状态按钮圆角 | `16px` |
| 卡片间距 | `12px` |

### 3.3 通用结构

- 顶部：餐次名称 + 时间段提示（可选）
- 中部：状态内容区
- 底部：操作区或摘要区

### 3.4 状态

| 状态 | 视觉 | 内容 | 交互 |
|------|------|------|------|
| empty | 浅灰背景 `#F7F8FA` 或极浅灰块，弱边界 | 显示 `+ 点击记录` 或 `告诉 AI 你吃了什么` | 点击进入输入或拍照记录 |
| pending | 白色背景，1px `#FFE7E1` 边框，必要时顶部可有浅橙提示条 | 显示 AI 解析出的食物内容 | 提供 `✓ 确认`、`✏️ 修改`、`✗ 取消` |
| recorded | 白色背景，正常卡片样式 | 显示食物列表、份量、总热量 | 点击进入详情页 |

### 3.5 ASCII 线框图

#### empty

```text

│ 早餐                         07:00-10:00 │
│                                     │
│          + 点击记录                  │
│      告诉 AI 你吃了什么              │
│                                     │
└─────────────────────────────────────┘
```

#### pending

```text
┌─────────────────────────────────────┐
│ 午餐                          AI 解析中 │
│ 鸡胸肉沙拉、玉米半根、无糖酸奶        │
│ 约 420 kcal                        │
│                                     │
│ [✓ 确认]   [✏️ 修改]   [✗ 取消]       │
└─────────────────────────────────────┘
```

#### recorded

```text
┌─────────────────────────────────────┐
│ 晚餐                              > │
│ • 三文鱼 150g                      │
│ • 西兰花 100g                      │
│ • 糙米饭 1 碗                      │
│ 总热量 568 kcal                    │
└─────────────────────────────────────┘
```

### 3.6 详细内容规范

- empty 状态文案优先展示行动引导，不展示无意义占位符。
- pending 状态中，AI 解析结果以自然语言分行展示，避免过度表格化。
- recorded 状态中，食物列表最多显示 3 行，超出用 `+2 项` 收起。
- 热量信息固定放在底部摘要区域，使用较大权重。
- 卡片点击区域：recorded 状态整卡可点；pending 状态优先点击底部按钮。

## 4. 数据记录卡片

### 4.1 描述

数据记录卡片用于数据页，承载体重、围度、睡眠、运动、饮水、排便等记录内容。组件需要保持统一骨架，便于用户快速识别，也便于 AI 自动生成同一页面中的多种数据卡片。

### 4.2 尺寸规格

| 项目 | 规格 |
|------|------|
| 卡片宽度 | 页面宽度减左右边距，自适应 |
| 推荐最小高度 | `116px` |
| 背景 | `#FFFFFF` |
| 圆角 | `12px` |
| 内边距 | `16px` |
| 标题字号 | `15px`~`16px`，SemiBold |
| 主数值字号 | `28px`~`36px`，Bold |
| 摘要字号 | `13px`~`14px` |
| 快捷按钮高度 | `28px`~`32px` |

### 4.3 通用结构

- 顶部：数据类型名称 + 日期/状态信息
- 中部：主内容区（大数字、图示、摘要）
- 底部：动作区或说明区

### 4.4 通用状态

| 状态 | 视觉 | 说明 |
|------|------|------|
| empty | 浅灰背景或白底虚位内容 | 尚未记录，鼓励开始记录 |
| pending | 白底 + 品牌浅色提示信息 | AI/系统暂存解析结果，等待用户确认 |
| recorded | 正常白底数据卡片 | 已确认记录，可查看详情或追加 |

### 4.5 类型定义

#### 4.5.1 体重卡片
- 主内容：大数字显示体重，如 `65.8 kg`
- 副信息：`比昨天 +0.3kg`
- recorded 态可附带小趋势箭头：上升/下降/持平
- 主数值居左更稳重，副信息放在下方

#### 4.5.2 围度卡片
- 展示腰围、臀围、胸围等一项或多项摘要
- 建议格式：`腰围 78cm｜臀围 95cm`
- 若只展示一项，则大数字突出，其他围度作为次级标签

#### 4.5.3 睡眠卡片
- 主内容：睡眠时长，如 `7h 35m`
- 次内容：`23:48 入睡｜07:23 起床`
- 质量标签：如 `睡得不错`、`偏浅眠`、`晚睡`
- 质量标签使用浅色胶囊，不抢主数值

#### 4.5.4 运动卡片
- 主内容可显示时长、消耗热量或步数摘要
- 示例：`步行 42 分钟`、`消耗 286 kcal`
- 可附带运动类型标签，如 `快走`、`力量训练`

#### 4.5.5 饮水卡片
- 核心内容：水杯插画 + 今日已喝/目标
- 示例：`1200 / 2000 ml`
- 底部快捷按钮：`+250ml`、`+500ml`
- 进度感要明显，可加浅色水位条

#### 4.5.6 排便卡片
- 用于日常健康习惯记录
- 主内容：今日次数、时间点、状态摘要
- 示例：`今日 1 次`、`07:40，正常`
- 风格保持轻松，不做医学化表达

## 5. 计划卡片

### 5.1 描述

计划卡片用于计划列表页，展示用户当前健康计划的目标、进度和状态。组件需要一眼能看出“在做什么”“做到哪里”“现在是什么状态”。

### 5.2 尺寸规格

| 项目 | 规格 |
|------|------|
| 卡片背景 | `#FFFFFF` |
| 圆角 | `12px` |
| 内边距 | `16px` |
| 最小高度 | `108px` |
| 计划名称字号 | `16px`，SemiBold |
| 进度条高度 | `8px` |
| 进度条圆角 | `999px` |
| 日期字号 | `13px` |
| 状态标签高度 | `24px` |

### 12.4 设计要求

- 插画风格要柔和、扁平、轻生活化。
- 文案最多两行，保持鼓励式语气。
- 若带按钮，按钮使用品牌色实心主按钮。
- 空状态整体垂直居中，不要贴近顶部。

### 12.5 ASCII 线框图

```text
              [ 200x200 插画 ]

       今天还没有记录哦，告诉 AI 你吃了什么吧

               [ 去记录 ]
```

### 12.6 Figma AI 生成描述

```text
生成一组移动端空状态组件，采用居中布局，顶部是 200x200px 柔和插画，中间是简短提示文案，下方是可选的主按钮。需要支持三个场景：1）未记录饮食：“今天还没有记录哦，告诉 AI 你吃了什么吧”；2）无计划：“还没有健康计划，让 AI 帮你制定一个？”；3）无数据：“暂无数据，开始记录后这里会显示趋势”。整体风格温暖、鼓励式、轻健康。
```

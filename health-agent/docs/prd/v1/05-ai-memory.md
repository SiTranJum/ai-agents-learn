# V1 - AI 记忆系统

> 模块：AI 记忆系统（三层记忆、提取、召回、质量管理）
> 优先级：P0（第二批开发）
> 依赖：01-用户体系
> 预计工时：2.5-3 周

---

## 1. 模块目标

建立 AI 记忆系统，让 AI 能"记住"用户的偏好、习惯和行为模式，实现个性化交互。

### 1.1 核心目标
- 用户能感知到"AI 记得我"
- AI 建议和对话能体现对用户的了解
- 记忆自动提取，无需用户主动告知
- 记忆在需要时自动召回
- 记忆质量可控，避免错误记忆

### 1.2 V1 标准
- 三层记忆架构搭建完成
- 4 个提取触发点实现
- 记忆质量检查机制
- 时间衰减召回策略
- 向量检索优化

## 2. 三层记忆架构

### 2.1 架构设计

| 层级 | 范围 | 内容 | 存储方式 | 生命周期 |
|------|------|------|---------|---------|
| 短期记忆 | 单次任务 | 当前对话上下文 | LLM 上下文窗口 | 任务结束即清空 |
| 中期记忆 | 7-30 天 | 近期行为模式、偏好变化 | 向量 DB + 结构化字段 | 时间衰减，30 天后归档 |
| 长期记忆 | 永久 | 稳定偏好、基础档案 | 结构化数据库 | 永久保存 |

### 2.2 短期记忆（Session Memory）

**定义：** 单次任务的上下文，存在于 LLM 的上下文窗口中。

**内容：**
- 当前对话历史（最近 5-10 轮）
- 当前任务状态（如正在创建计划）
- 临时变量（如用户刚说的食物名称）

**实现：**
- LangGraph 的 State 管理
- 任务结束后清空

**示例：**
```
用户："我今天中午吃了鸡胸肉"
AI："好的，记录了 100g 鸡胸肉。还吃了其他东西吗？"
用户："还有一碗米饭"
AI："明白，已记录鸡胸肉 + 米饭"
```
（"鸡胸肉"在短期记忆中，AI 知道"还有"指的是同一餐）

### 2.3 中期记忆（Behavioral Memory）

**定义：** 近期（7-30 天）的行为模式和偏好变化。

**内容：**
- 近期饮食偏好（最近常吃什么）
- 近期行为模式（记录时间习惯、运动频率）
- AI 建议的采纳/拒绝记录
- 用户纠正记录

**存储方式：**
- 向量数据库（Supabase pgvector）：用于语义检索
- 结构化字段：用于精确查询

**数据结构：**
```json
{
  "id": "uuid",
  "user_id": "uuid",
  "memory_type": "food_preference",
  "content": "用户最近一周经常吃鸡胸肉，每次约 100-150g",
  "embedding": [0.1, 0.2, ...],
  "metadata": {
    "food": "鸡胸肉",
    "frequency": 5,
    "date_range": "2026-04-20 ~ 2026-04-27"
  },
  "quality_score": 85,
  "created_at": "2026-04-27",
  "last_accessed": "2026-04-27",
  "access_count": 3,
  "time_decay_factor": 1.0
}
```

**生命周期：**
- 基于时间衰减机制，权重逐渐降低
- 30 天后归档但不删除
- 重要记忆可提升为长期记忆

### 2.4 长期记忆（Profile Memory）

**定义：** 稳定的用户档案和长期偏好。

**内容：**
- 用户基础档案（性别、年龄、目标等）
- 饮食偏好和限制（素食、过敏、忌口）
- 基础疾病信息
- 长期稳定的偏好（如"不喜欢香菜"）

**存储方式：**
- 结构化数据库（PostgreSQL）
- 已在用户体系模块定义

**特点：**
- 永久保存
- 每次任务开始时加载
- 用户可手动编辑

## 3. 记忆提取机制

### 3.1 提取触发点

V1 实现 4 个提取触发点：

| 触发点 | 触发时机 | 提取内容 | 示例 |
|--------|---------|---------|------|
| 饮食记录 | 用户记录饮食后 | 食物偏好、份量习惯 | 用户连续 3 天吃鸡胸肉 → 提取"喜欢鸡胸肉" |
| 用户纠正 | 用户纠正 AI 输出 | 纠正内容、偏好调整 | AI 说"一碗米饭约 200g"，用户改成 150g → 提取"用户的一碗米饭约 150g" |
| 建议反馈 | 用户采纳/忽略 AI 建议 | 建议偏好 | AI 建议"晚餐吃鱼"，用户采纳 → 提取"用户接受鱼类建议" |
| 计划执行 | 计划执行过程中 | 执行习惯、困难点 | 用户连续 3 天未完成计划 → 提取"计划目标可能过高" |

### 3.2 提取流程

```
触发点发生
    ↓
[后台异步任务] 分析行为数据
    ↓
[LLM] 提取记忆片段
    ↓
[质量检查] 评分和验证
    ↓
通过 → 生成记忆对象：
  - content: 自然语言描述
  - embedding: 向量化
  - metadata: 结构化字段
  - quality_score: 质量评分
    ↓
存入中期记忆（Supabase pgvector + PostgreSQL）
```

### 3.3 记忆质量检查

**检查时机：** 记忆提取后、存储前

**质量维度：**

| 维度 | 权重 | 说明 | 评分标准 |
|------|------|------|---------|
| 相关性 | 30% | 是否与健康管理相关 | 0-100 分 |
| 准确性 | 40% | 是否有明确的事实依据 | 0-100 分 |
| 可操作性 | 20% | 是否能用于后续建议 | 0-100 分 |
| 非冗余性 | 10% | 是否与已有记忆重复 | 0-100 分 |

**综合评分公式：**
```
质量分 = 相关性×0.3 + 准确性×0.4 + 可操作性×0.2 + 非冗余性×0.1
```

**处理策略：**
- 质量分 >= 80：直接存储
- 质量分 60-79：标记为"待验证"，存储但降低召回权重
- 质量分 < 60：不存储，记录日志

**质量检查 Prompt 示例：**
```python
quality_check_prompt = """
评估以下记忆片段的质量：

记忆内容：{memory_content}
用户档案：{user_profile}
已有记忆：{existing_memories}

请从以下维度评分（0-100）：
1. 相关性：是否与用户的健康管理目标相关？
2. 准确性：是否基于明确的事实和数据？
3. 可操作性：是否能用于生成个性化建议？
4. 非冗余性：是否与已有记忆重复？

输出格式：
{
  "relevance": 85,
  "accuracy": 90,
  "actionability": 80,
  "uniqueness": 75,
  "overall_score": 83,
  "reason": "基于用户连续 5 天的饮食记录，准确反映了食物偏好"
}
"""
```

## 4. 记忆召回机制

### 4.1 召回优先级

```
任务开始
    ↓
[1] 加载长期记忆（全量）
    ↓
[2] 根据意图召回中期记忆（向量检索 + 多因子评分）
    ↓
[3] 根据时间/上下文召回相关中期记忆
    ↓
组装成 LLM Prompt 的一部分
```

### 4.2 时间衰减机制

**衰减公式：**
```
时间衰减系数 = 基础系数 × 访问频率加成

基础系数（基于创建时间）：
- 7 天内：1.0（满分）
- 7-14 天：0.9
- 14-21 天：0.8
- 21-30 天：0.7
- 30 天后：0.5（归档状态）

访问频率加成：
- 访问次数 >= 5：+0.1
- 访问次数 >= 10：+0.2
```

**示例：**
```
记忆 A：创建于 10 天前，访问 3 次
  → 基础系数 0.9，无加成
  → 最终系数 0.9

记忆 B：创建于 10 天前，访问 8 次
  → 基础系数 0.9，加成 +0.1
  → 最终系数 1.0
```

### 4.3 多因子召回评分

**召回评分公式：**
```
最终分数 = 向量相似度 × 时间衰减系数 × 类型权重 × 质量系数

其中：
- 向量相似度：0-1（Supabase pgvector 检索结果）
- 时间衰减系数：0.5-1.0
- 类型权重：0.6-1.0（根据当前意图）
- 质量系数：quality_score / 100
```

**类型权重配置：**
```python
type_weights = {
  "record_diet": {
    "food_preference": 1.0,
    "portion_habit": 0.9,
    "behavior_pattern": 0.7,
    "suggestion_feedback": 0.6
  },
  "create_plan": {
    "behavior_pattern": 1.0,
    "suggestion_feedback": 0.9,
    "food_preference": 0.8,
    "portion_habit": 0.6
  },
  "general_advice": {
    "food_preference": 1.0,
    "behavior_pattern": 0.9,
    "suggestion_feedback": 0.8,
    "portion_habit": 0.7
  }
}
```

### 4.4 召回策略

**长期记忆：**
- 每次任务开始时全量加载
- 作为 System Prompt 的一部分

**中期记忆：**
- 向量检索 Top 10 候选
- 多因子评分排序
- 取 Top 3 最终结果
- 过滤质量分 < 60 的记忆

### 4.5 召回示例

**场景：** 用户说"帮我记录今天的午餐"

**召回过程：**
```
1. 向量检索 Top 10：
   - 记忆 A：相似度 0.92，创建 5 天前，quality 85
   - 记忆 B：相似度 0.88，创建 15 天前，quality 90
   - 记忆 C：相似度 0.85，创建 3 天前，quality 75
   ...

2. 多因子评分：
   - 记忆 A：0.92 × 1.0 × 1.0 × 0.85 = 0.782
   - 记忆 B：0.88 × 0.9 × 0.9 × 0.90 = 0.643
   - 记忆 C：0.85 × 1.0 × 0.9 × 0.75 = 0.574

3. 排序后 Top 3：
   - 记忆 A（0.782）
   - 记忆 B（0.643）
   - 记忆 C（0.574）
```

**召回的记忆：**
```
[长期记忆]
- 用户性别：男，年龄 36，目标：减脂
- 饮食限制：无
- 不喜欢的食物：香菜、芹菜

[中期记忆 - Top 3]
1. 用户最近一周经常吃鸡胸肉（评分 0.782）
2. 用户的"一碗米饭"约 150g（评分 0.643）
3. 用户午餐时间通常在 12:00-13:00（评分 0.574）
```

**组装后的 Prompt：**
```
System: 你是健康管家 AI。用户档案：男，36 岁，目标减脂。不喜欢香菜、芹菜。

最近记忆：
- 用户最近一周经常吃鸡胸肉
- 用户的"一碗米饭"约 150g
- 用户午餐时间通常在 12:00-13:00

User: 帮我记录今天的午餐
```

## 5. 记忆合并与摘要（可选）

**触发条件：**
- 每周一次自动检查
- 或累积 20 条新记忆时触发

**合并流程：**
```
1. 检测相似记忆（向量相似度 > 0.85）
    ↓
2. 用 LLM 将相似记忆合并成摘要
    ↓
3. 原始记忆标记为"已合并"，保留但不参与召回
    ↓
4. 摘要记忆参与召回
```

**摘要结构：**
```json
{
  "type": "食物偏好摘要",
  "content": "用户喜欢吃鸡胸肉，最近两周吃了 8 次，通常在午餐或晚餐，份量约 100-150g",
  "source_memories": ["memory_id_1", "memory_id_2", ...],
  "quality_score": 92,
  "confidence": 0.92,
  "created_at": "2026-04-27",
  "last_updated": "2026-04-27"
}
```

**合并 Prompt 示例：**
```python
merge_prompt = """
将以下相似的记忆片段合并成一条摘要：

记忆 1: 用户 2026-04-23 午餐吃了鸡胸肉 100g
记忆 2: 用户 2026-04-24 晚餐吃了鸡胸肉 150g

记忆 4: 用户 2026-04-26 晚餐吃了鸡胸肉 120g
记忆 5: 用户 2026-04-27 午餐吃了鸡胸肉 100g

请生成一条结构化摘要，包含：
- 食物名称
- 频率统计
- 时间模式（哪些餐次）

输出格式：
{

  "metadata": {...}
}
"""
```

## 6. 测试要点

### 6.1 记忆提取测试
- 各触发点是否正确提取
- 质量检查是否有效过滤低质量记忆
- 异步提取不阻塞主流程

### 6.2 记忆召回测试
- 向量检索准确性
- 时间衰减是否生效
- 多因子评分排序正确性
- 类型权重是否影响召回结果

### 6.3 质量管理测试
- 质量评分准确性
- 低质量记忆是否被过滤
- 待验证记忆的处理

### 6.4 时间衰减测试
- 不同时间段的衰减系数
- 访问频率加成是否生效
- 30 天后归档机制

### 6.5 记忆合并测试（可选）
- 相似记忆检测准确性
- 合并后摘要质量
- 原始记忆是否正确标记

### 6.6 用户感知测试
- AI 是否体现"记得用户"
- 个性化建议是否准确
- 记忆召回是否及时

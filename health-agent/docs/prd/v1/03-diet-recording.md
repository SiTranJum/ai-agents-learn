# V1 - 饮食记录与管理

> 模块：饮食记录（文本+拍照+营养查询+记录管理）
> 优先级：P0（第三批开发）
> 依赖：02-应用框架、06-RAG 知识库
> 预计工时：2-2.5 周

---

## 1. 模块目标

实现完整的饮食记录链路：输入 → 识别/解析 → 确认 → 保存 → 汇总展示。支持文本和拍照两种输入方式。

### 1.1 核心目标
- 用户能通过自然语言快速记录饮食
- 用户能通过拍照识别食物并记录
- 系统自动计算热量和营养成分
- 用户能查看、编辑、删除饮食记录
- 用户能查看每日营养摄入汇总

### 1.2 V1 标准
- 文本记录：解析准确率 > 85%
- 拍照识别：常见食物识别率 > 70%，允许用户纠正
- 营养数据：覆盖常见中国食物，查不到的用 AI 估算
- 不追求复杂混合菜品的极高精度

## 2. 功能清单

### 2.1 自然语言饮食记录

#### 2.1.1 输入方式
- Command Bar 输入（主要方式）
- 饮食记录页面的输入框

#### 2.1.2 AI 解析能力

**基础解析：**
- 识别食物名称："鸡胸肉"、"米饭"、"兰州拉面"
- 识别份量："100g"、"一碗"、"一盘"、"半份"
- 识别餐次：根据时间自动判断或用户指定（"今天中午"、"早餐"）
- 识别多个食物："一碗米饭加一个鸡腿"

**进阶解析：**
- 俗称识别："牛肉面" = "兰州拉面"
- 复合食物："兰州拉面加个蛋" → 拉面 + 鸡蛋
- 模糊份量估算："一碗" → 约 300g（米饭）
- 烹饪方式识别："水煮鸡胸肉" vs "油炸鸡胸肉"（影响热量）

#### 2.1.3 解析流程

```
用户输入："今天中午吃了一碗兰州拉面加个蛋"
    ↓
[LLM 解析] 结构化输出
    ↓
{
  "meal_type": "lunch",
  "foods": [
    {"name": "兰州拉面", "amount": 1, "unit": "碗", "cooking": "煮"},
    {"name": "鸡蛋", "amount": 1, "unit": "个", "cooking": "煮"}
  ]
}
    ↓
[营养查询] 从知识库/API 查询每个食物的营养数据
    ↓
{
  "foods": [
    {"name": "兰州拉面", "amount": 300, "unit": "g", "calories": 450, "carbs": 65, "protein": 15, "fat": 12},
    {"name": "鸡蛋", "amount": 50, "unit": "g", "calories": 78, "carbs": 0.6, "protein": 6.5, "fat": 5.3}
  ],
  "total": {"calories": 528, "carbs": 65.6, "protein": 21.5, "fat": 17.3}
}
    ↓
[展示结果卡片] 根据交互模式展示
    ↓
[用户确认/修改] → 保存到数据库
```

#### 2.1.4 用户修改能力
- 修改食物名称
- 修改份量
- 删除某个食物
- 添加遗漏的食物
- 修改餐次

### 2.2 拍照识别

#### 2.2.1 入口
- Command Bar 旁的相机图标
- 饮食记录页面的"拍照记录"按钮

#### 2.2.2 拍照流程

```
用户上传/拍摄食物照片
    ↓
[多模态 LLM] 识别食物
    ↓
返回识别结果：
{
  "foods": [
    {"name": "红烧肉", "confidence": 0.85, "estimated_amount": "一份"},
    {"name": "米饭", "confidence": 0.92, "estimated_amount": "一碗"}
  ]
}
    ↓
[营养查询] 查询营养数据
    ↓
[展示结果卡片] 用户确认/修改
    ↓
保存到数据库
```

#### 2.2.3 识别结果展示
- 显示识别出的食物列表
- 每个食物显示置信度（高/中/低）
- 低置信度的食物标记提示用户确认
- 用户可修改识别结果

#### 2.2.4 V1 拍照识别标准
- 支持单张图片上传
- 识别常见中国食物（家常菜、快餐、水果等）
- 70-80% 准确率
- 允许用户纠正识别结果
- 不追求：多菜品精确分割、份量精确估算

---

### 2.3 食物营养查询

#### 2.3.1 查询方式
- Command Bar 输入："鸡胸肉的热量"
- 饮食记录页面的搜索框

#### 2.3.2 数据源策略（优先级从高到低）

| 优先级 | 数据源 | 说明 |
|--------|--------|------|
| 1 | 本地知识库 | RAG 向量检索，覆盖常见食物 |
| 2 | 第三方 API | FatSecret / Open Food Facts |
| 3 | AI 估算 | LLM 实时估算，标记"AI 估算" |

#### 2.3.3 查询结果展示

**标准展示：**
- 食物名称
- 每 100g 热量
- 三大营养素（碳水/蛋白质/脂肪）
- 数据来源标记（知识库/API/AI 估算）

**学习模式额外展示：**
- 微量元素（如有）
- 食物特点说明
- 适合人群

#### 2.3.4 份量换算
- 支持常见单位换算："一碗米饭" ≈ 200g
- 常见换算表：

| 食物 | 单位 | 约等于 |
|------|------|--------|
| 米饭 | 一碗 | 200g |
| 面条 | 一碗 | 300g |
| 鸡蛋 | 一个 | 50g |
| 苹果 | 一个 | 200g |
| 牛奶 | 一杯 | 250ml |

### 2.4 饮食记录管理

#### 2.4.1 记录列表页

**页面路由：** `/diet`

**默认视图：** 今日饮食

**展示方式：**
- 按餐次分组：早餐、午餐、晚餐、加餐
- 每个餐次以卡片形式展示
- 卡片显示：食物列表、该餐总热量
- 未记录的餐次显示"+ 点击记录"提示

**卡片状态：**
- empty（未记录）：显示"+ 点击记录"提示
- pending（待确认）：显示内容 + [✓ 确认] [✏️ 修改] [✗ 取消] 按钮
- recorded（已记录）：显示内容，无操作按钮

**日期切换：**
- 顶部日期选择器
- 左右箭头切换前后一天
- 点击日期打开日历选择

#### 2.4.2 页面内卡片交互

**记录饮食流程：**
```
用户在输入框输入："午餐吃了鸡胸肉"
    ↓
AI 解析 → 识别为午餐记录
    ↓
午餐卡片状态变为 pending
    ↓
卡片内容更新：显示"鸡胸肉 165kcal"
    ↓
卡片底部出现：[✓ 确认] [✏️ 修改] [✗ 取消]
    ↓
用户点击 [✓ 确认] → 保存，卡片状态变为 recorded
```

#### 2.4.3 修改记录的两种方式

**方式 1：详情页修改（表单式）**

适用场景：大幅修改、改多个字段、不确定怎么说

流程：
```
用户点击卡片上的 [✏️ 修改]
    ↓
跳转到详情页 /diet/record/{id}/edit
    ↓
表单预填充当前记录：
  - 食物名称：[鸡胸肉]
  - 份量：[100] g
  - 餐次：[午餐]
    ↓
用户修改后点 [保存]
    ↓
返回饮食页面，卡片更新
```

**方式 2：对话式修改（AI 理解）**

适用场景：小幅修改、只改一个字段、用户习惯对话

流程：
```
用户在输入框说："把午餐的鸡胸肉改成150g"
    ↓
AI 解析意图：
{
  "intent": "modify_diet",
  "target": {"meal": "lunch", "food": "鸡胸肉"},
  "change": {"amount": 150, "unit": "g"}
}
    ↓
前端定位到午餐卡片中的"鸡胸肉"记录
    ↓
更新卡片内容：鸡胸肉 150g 248kcal
    ↓
卡片重新显示 [✓ 确认] [✏️ 修改] [✗ 取消]
```

**处理歧义：**
- 如果午餐有多条鸡胸肉 → AI 追问："午餐有两条鸡胸肉记录，你要改哪一条？"
- 如果无法定位 → 提示："我没找到这条记录，你可以点击修改按钮进入详情页"

**支持的修改指令：**
- 修改份量："把午餐的鸡胸肉改成150g"
- 修改食物："把午餐的鸡胸肉改成牛肉"
- 删除食物："删除午餐的鸡胸肉"
- 添加食物："午餐再加一碗米饭"

#### 2.4.4 删除记录

**删除整餐：**
```
用户说："删除午餐记录"
    ↓
AI 识别为删除操作（置信度检查）
    ↓
弹出确认对话框：
  "确定删除午餐记录吗？删除后无法恢复"
  [确定删除] [取消]
    ↓
用户确认 → 删除记录，卡片状态变为 empty
```

**删除单个食物：**
- 对话式："删除午餐的鸡胸肉"
- 详情页：进入编辑页面，删除对应食物

#### 2.4.5 今日摄入汇总

**展示在饮食记录页顶部：**
- 今日总热量 / 目标热量（进度条）
- 三大营养素摄入 / 目标（进度条）
- 各餐次热量占比（简单柱状图）

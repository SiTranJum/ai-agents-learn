# 项目结构重组与文档体系建立计划

## 背景

课程学习阶段（courses/01-27）已完成，进入实战 Agent 开发阶段。需要：

1. 为健康管家 Agent 建立独立的项目结构，与 courses/ 分离
2. 建立文档驱动开发流程：总 PRD → 子 PRD → 技术方案 Spec → 开发
3. 文档与代码分离，前后端代码各自独立

## 项目结构

```
agent/
├── courses/                    # 学习课程（保持不变）
├── health-agent/               # 健康管家 Agent 项目根目录
│   ├── docs/                   # 项目文档（文档与代码分离）
│   │   ├── plans/              # 项目计划与开发计划
│   │   ├── prd/                # 产品需求文档
│   │   │   ├── 00-master-prd.md
│   │   │   ├── 01-core-conversation.md
│   │   │   ├── 02-diet-recording.md
│   │   │   └── ...
│   │   ├── specs/              # 技术方案文档
│   │   │   ├── 00-architecture.md
│   │   │   ├── 01-tech-stack.md
│   │   │   └── ...
│   │   └── adr/                # 架构决策记录
│   ├── backend/                # 后端项目（等架构 Spec 确定后创建）
│   └── frontend/               # 前端项目（等架构 Spec 确定后创建）
├── CLAUDE.md
├── PLAN.md
└── .venv/
```

### 设计原则

1. **health-agent/** 作为独立项目根目录，与 courses/ 平级
2. **docs/** 文档与代码完全分离，包含四层：
   - `plans/`：项目管理计划和开发计划
   - `prd/`：产品需求文档，00 开头是总 PRD，01-xx 是子 PRD
   - `specs/`：技术方案文档，00 开头是总架构，01-xx 是各模块技术方案
   - `adr/`：架构决策记录，记录重要技术选型的原因和权衡
3. **backend/** 和 **frontend/** 前后端分离，独立项目包，等架构 Spec 确定后再创建

## 文档体系

### 编写顺序

```
总 PRD (00-master-prd.md)
  ↓
子 PRD (01-xx.md)
  ↓
总体架构 Spec (00-architecture.md)
  ↓
各模块技术 Spec (01-xx.md)
  ↓
开发实施
```

### PRD 体系

**总 PRD（00-master-prd.md）** 需包含：
- 用户故事与用户旅程
- 功能优先级矩阵（P0/P1/P2）
- 数据流图与交互流程图
- 成功指标
- 风险与假设

**子 PRD 拆分维度：**

| 编号 | 模块 | 核心内容 |
|------|------|----------|
| 01 | 核心对话能力 | 自然语言理解、多轮对话、意图识别 |
| 02 | 饮食记录 | 文字解析、拍照识别、食物数据库查询 |
| 03 | 记忆系统 | 短期/中期/长期记忆、记忆召回、用户画像 |
| 04 | RAG 知识库 | 营养知识库、健康建议库、向量检索 |
| 05 | 身体数据管理 | 体重/围度记录、数据可视化、趋势分析 |
| 06 | 主动提醒 | 提醒时机判断、推送策略、用户偏好学习 |

### 技术 Spec 体系

每个子 PRD 对应一个技术 Spec，包含：
- 技术方案选型
- 数据模型设计
- API 接口设计
- 关键算法/流程
- 性能指标
- 测试策略

# 课程 9：给 Agent 加上记忆能力

## 学习目标
1. 理解为什么 Agent 需要记忆系统
2. 掌握短期记忆（对话历史）的实现
3. 掌握长期记忆（向量存储）的基础实现
4. 将记忆系统整合到课程 8 的 Agent 中
5. 理解记忆召回的时机和策略

## 为什么这节课重要？

课程 8 做的 Agent 有个致命问题：**它不记得之前的对话**。

想象一下：
```
用户：我今天早上吃了两个鸡蛋
Agent：好的，记录了早餐：2 个鸡蛋，约 140 卡路里

用户：我的早餐热量是多少？
Agent：抱歉，我不知道你今天吃了什么早餐
```

这就是没有记忆的 Agent。课程 9 要解决这个问题。

## 课程内容

### 1. Agent 记忆系统概述（15 分钟）
- 为什么需要记忆？
- 记忆的两种类型：
  - **短期记忆**：当前对话的上下文（对话历史）
  - **长期记忆**：跨会话的持久化信息（用户偏好、历史数据）
- 记忆召回的时机

### 2. 短期记忆实现（30 分钟）
- 对话历史管理
- 上下文窗口限制
- 历史压缩策略
- 代码实现：`ConversationMemory` 类

### 3. 长期记忆基础（30 分钟）
- 什么需要长期记忆？
- 简单的持久化方案（JSON 文件）
- 向量存储预告（课程 16 深入）
- 代码实现：`LongTermMemory` 类

### 4. 整合到 Agent（30 分钟）
- 修改课程 8 的 Agent
- 添加记忆模块
- 记忆召回逻辑
- 完整测试

### 5. 记忆系统的优化方向（10 分钟）
- 记忆的优先级
- 记忆的遗忘机制
- 向量检索（后续课程）

## 实践任务

在课程 8 的健康管家 Agent 基础上，添加记忆能力：

1. **短期记忆**：
   - 记住当前对话中记录的饮食
   - 能回答"我今天吃了什么"
   - 能回答"我今天摄入了多少卡路里"

2. **长期记忆**：
   - 记住用户的基本信息（身高、体重、目标）
   - 记住用户的饮食偏好
   - 下次对话时能调用这些信息

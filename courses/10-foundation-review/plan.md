# 课程 10：基础阶段总复习 + 小测验

## 学习目标
1. 回顾课程 1-8 的核心知识点
2. 梳理 AI Agent 的完整知识体系
3. 通过测验检验学习效果
4. 识别薄弱环节，针对性巩固

## 为什么这节课重要？

前面 8 节课学了很多内容，现在需要：
- **串联知识**：把零散的知识点连成体系
- **查漏补缺**：发现理解不到位的地方
- **实战检验**：通过测验验证掌握程度
- **承上启下**：为进阶课程（阶段三）做准备

## 课程内容

### 1. 知识体系回顾（30 分钟）

#### 1.1 LLM 基础（课程 1）
- LLM 的核心原理
- Token、Temperature、上下文窗口
- OpenAI SDK 的使用
- 多轮对话实现

#### 1.2 Prompt Engineering（课程 2）
- Prompt 的组成（System + User）
- 五大原则：清晰具体、上下文、输出格式、Few-shot、CoT
- 如何写出高质量的 Prompt

#### 1.3 Tool Use（课程 3）
- Function Calling 原理
- 工具定义（JSON Schema）
- 多轮调用流程
- 用户体验设计

#### 1.4 Agent 核心概念（课程 4-7）
- 什么是 Agent？与普通 LLM 调用的区别
- Agent 循环：感知 → 思考 → 行动
- Memory：短期、中期、长期记忆
- Planning：任务拆解与规划

#### 1.5 完整 Agent 实现（课程 8）
- Agent 架构设计
- 组件整合
- 实战：健康管家 Agent

### 2. 知识图谱梳理（20 分钟）

绘制知识图谱，理清各概念之间的关系：

```
LLM（基础）
  ├─ Prompt Engineering（如何沟通）
  ├─ Tool Use（如何行动）
  └─ Agent（自主循环）
       ├─ Agent Loop（核心机制）
       ├─ Memory（记忆能力）
       └─ Planning（规划能力）
```

### 3. 小测验（40 分钟）

#### 3.1 理论题（10 题）
- 选择题：测试概念理解
- 判断题：测试常见误区
- 简答题：测试深度理解

#### 3.2 实践题（3 题）
- 编写 Prompt
- 定义工具（JSON Schema）
- 实现简单 Agent

### 4. 答案解析 + 知识巩固（30 分钟）
- 逐题讲解
- 常见错误分析
- 薄弱环节补充

### 5. 进阶路线预告（10 分钟）
- 阶段三：进阶能力课程
- RAG、Multi-Agent、框架选型
- 健康管家项目实战

## 测验说明

### 测验形式
- **开卷测验**：可以查阅课程资料
- **实践为主**：重点考察动手能力
- **无时间限制**：理解比速度重要

### 评分标准
- 理论题：每题 5 分，共 50 分
- 实践题：每题 10-20 分，共 50 分
- 总分 100 分，60 分及格

### 测验目的
- **不是考试**：目的是检验学习效果，不是打分
- **查漏补缺**：发现薄弱环节，针对性学习
- **建立信心**：验证自己的进步

## 预计时长
- 知识回顾：约 50 分钟
- 测验：约 60 分钟
- 答案解析：约 30 分钟
- 总计：约 2.5 小时

## 完成标准
- 完成所有测验题目
- 理解答案解析
- 识别出自己的薄弱环节
- 准备好进入阶段三

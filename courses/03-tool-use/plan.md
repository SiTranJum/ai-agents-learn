# 课程 3：Tool Use（工具调用）

## 学习目标
1. 理解 Tool Use 的核心概念和工作原理
2. 掌握如何定义和使用工具（Function Calling）
3. 学会处理工具调用的结果和错误
4. 了解工具调用在 Agent 中的应用场景
5. 实践：构建一个能调用外部工具的 AI Agent

## 为什么这节课重要？
Tool Use 是 Agent 和普通聊天机器人的核心区别：
- **普通 LLM**：只能生成文本，无法与外部世界交互
- **带 Tool Use 的 Agent**：可以查询数据库、调用 API、执行计算、操作文件等

对于我们的健康管家产品：
- 查询食物营养数据库
- 计算每日营养摄入
- 记录用户数据到数据库
- 生成图表和报告
- 设置提醒推送

这些都需要 Tool Use 能力。

## 课程内容

### 1. Tool Use 基础概念（20 分钟）
- 什么是 Tool Use / Function Calling
- LLM 如何"决定"调用工具
- 工具定义的结构（JSON Schema）
- 工具调用的完整流程

### 2. 定义和使用工具（30 分钟）
- 工具定义的规范（name, description, parameters）
- 参数类型和验证
- 在 API 调用中传递工具定义
- 解析 LLM 返回的工具调用请求

### 3. 执行工具和返回结果（30 分钟）
- 执行实际的工具函数
- 将结果返回给 LLM
- 多轮对话中的工具调用
- 错误处理和重试机制

### 4. 实践：构建天气查询 Agent（40 分钟）
- 定义天气查询工具
- 实现工具函数（模拟 API 调用）
- 构建完整的对话循环
- 处理多轮工具调用

### 5. 进阶话题（20 分钟）
- 并行工具调用
- 工具调用的最佳实践
- 常见问题和调试技巧
- 与健康管家产品的结合点

## 实践任务
1. **基础练习**：定义一个计算器工具，支持加减乘除
2. **进阶练习**：构建一个能查询天气和设置提醒的 Agent
3. **挑战任务**：为健康管家设计 3-5 个核心工具（食物查询、营养计算等）

- 讲解 + 演示：约 90 分钟
- 练习 + 讨论：约 60 分钟
- 总计：2.5 小时

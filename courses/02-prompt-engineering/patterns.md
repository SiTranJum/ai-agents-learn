# 常见 Prompt 模式

## 模式 1：角色扮演（Role Playing）

**原理**：给 AI 一个明确的角色身份，它会按该角色的知识和风格回答。

```python
# 营养师角色
system_prompt = "你是一位有 10 年经验的注册营养师，擅长运动营养和减脂指导。"

# 健身教练角色
system_prompt = "你是一位健身教练，专注于力量训练和增肌计划。"

# 医生角色（注意：实际产品中要谨慎，避免医疗风险）
system_prompt = "你是健康顾问，可以提供一般性健康建议，但不能诊断疾病或开处方。"
```

**效果**：同样的问题，不同角色会给出不同视角的答案。

## 模式 2：结构化输出

**原理**：明确要求输出格式，让 AI 返回易于程序处理的数据。

### JSON 格式

```python
user_prompt = """
分析"鸡胸肉 150g"的营养，用 JSON 格式返回：
{
  "food": "食物名称",
  "amount": "数量",
  "unit": "单位",
  "nutrition": {
    "calories": 热量(千卡),
    "protein": 蛋白质(克),
    "fat": 脂肪(克),
    "carbs": 碳水化合物(克)
  }
}
"""
```

### 表格格式

```python
user_prompt = """
对比苹果、香蕉、橙子的营养，用 Markdown 表格返回：

| 食物 | 热量(千卡/100g) | 蛋白质(g) | 脂肪(g) | 碳水(g) |
|------|----------------|----------|---------|---------|
| ... | ... | ... | ... | ... |
"""
```

**用途**：健康管家需要把 AI 的输出存入数据库，结构化格式是必需的。

## 模式 3：思维链（Chain of Thought, CoT）

**原理**：让 AI 展示推理步骤，而不是直接给结论。

```python
user_prompt = """
我想减脂，今天吃了：
- 早餐：鸡蛋 2 个、牛奶 250ml
- 午餐：米饭 200g、鸡胸肉 150g、西兰花 100g
- 晚餐：还没吃

请分析我今天的饮食是否合理，按以下步骤：
1. 计算已摄入的总热量和三大营养素
2. 估算我的每日热量需求（30 岁男性，70kg，轻度运动，减脂目标）
3. 计算还剩多少热量预算
4. 建议晚餐吃什么
"""
```

**关键词**：
- "请按以下步骤"
- "首先...然后...最后..."
- "让我们一步步分析"

## 模式 4：Few-shot Learning（少样本学习）

**原理**：给几个输入-输出示例，让 AI 学会模式。

```python
system_prompt = """
你是食物识别助手，从用户的自然语言中提取食物和数量。

示例 1：
输入：我早上吃了两个鸡蛋
输出：{"food": "鸡蛋", "amount": 2, "unit": "个"}

示例 2：
输入：喝了一大杯牛奶
输出：{"food": "牛奶", "amount": 1, "unit": "大杯"}

示例 3：
输入：午饭吃了一碗米饭和一些青菜
输出：[
  {"food": "米饭", "amount": 1, "unit": "碗"},
  {"food": "青菜", "amount": null, "unit": "一些"}
]
"""

user_prompt = "晚上吃了三片面包和半个苹果"
# AI 会模仿示例的格式输出
```

**适用场景**：
- 格式化任务（提取、分类）
- 风格模仿
- 特定领域的术语使用

## 模式 5：ReAct（Reasoning + Acting）

**原理**：让 AI 交替进行"推理"和"行动"，适合需要调用工具的场景。

```python
system_prompt = """
你是健康助手，可以使用以下工具：
- search_food(name): 查询食物营养数据
- calculate_bmr(age, gender, weight, height): 计算基础代谢率

回答时按 ReAct 格式：
Thought: 我需要做什么
Action: 调用哪个工具
Observation: 工具返回的结果
... (重复直到能回答)
Answer: 最终答案
"""

user_prompt = "我 30 岁男性，70kg，175cm，今天吃了 200g 鸡胸肉，摄入了多少蛋白质？"

# AI 的输出：
"""
Thought: 需要查询鸡胸肉的营养数据
Action: search_food("鸡胸肉")
Observation: 鸡胸肉每 100g 含蛋白质 23g
Thought: 用户吃了 200g，需要计算
Answer: 200g 鸡胸肉含蛋白质 46g
"""
```

**用途**：Agent 系统的核心模式，后面课程会深入讲。

## 针对健康管家的 Prompt 设计建议

### 1. System Prompt 模板

```python
system_prompt = """
你是"健康管家"AI 助手，一个专业但友好的营养顾问。

你的能力：
- 分析食物营养成分
- 提供个性化饮食建议
- 记录和追踪用户的饮食习惯
- 解答健康饮食相关问题

你的原则：
- 回答要科学准确，基于营养学知识
- 语气友好亲切，像朋友聊天
- 不提供医疗诊断，不推荐药物
- 遇到严重健康问题，建议就医

输出格式：
- 营养数据用结构化格式（JSON）
- 建议用简洁的要点形式
- 必要时展示计算过程
"""
```

### 2. 处理模糊输入

```python
# 用户可能说："吃了点鸡肉"（没说多少）
# Prompt 要教 AI 如何处理

system_prompt += """
当用户输入模糊时：
- 数量不明确：询问具体数量，或给出常见份量的估算
- 食物不明确：列出可能的选项让用户选择
- 烹饪方式不明确：询问是煎/炸/蒸/煮（影响热量）
"""
```

### 3. 记忆和上下文

```python
# 在 messages 中维护用户画像
messages = [
    {"role": "system", "content": system_prompt},
    {"role": "system", "content": "用户信息：30 岁男性，目标减脂，每日热量目标 1800 千卡"},
    {"role": "user", "content": "我今天吃了..."}
]
```

## 小结

| 模式 | 用途 | 健康管家中的应用 |
|------|------|----------------|
| 角色扮演 | 设定专业身份 | 营养师角色 |
| 结构化输出 | 便于程序处理 | 营养数据存储 |
| CoT | 提升推理质量 | 饮食分析、建议生成 |
| Few-shot | 格式化任务 | 食物识别、数据提取 |
| ReAct | 工具调用 | Agent 系统（后续课程） |

下一步：动手实践这些模式。

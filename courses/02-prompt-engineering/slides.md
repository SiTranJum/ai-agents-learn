# 课件：Prompt Engineering（提示词工程）

## 1. 什么是 Prompt Engineering？

**Prompt = 你给 AI 的指令**

类比 Java：
```java
// 你写代码时，逻辑是明确的
public String getAdvice(String food) {
    if (calories > 500) return "热量偏高";
    return "热量适中";
}

// 用 LLM 时，"逻辑"是通过 prompt 描述的
prompt = "你是营养师，分析食物热量，超过 500 卡说'热量偏高'，否则说'热量适中'"
```

**Prompt Engineering = 如何写出让 AI 准确理解并执行的指令**

就像写需求文档：
- 写得模糊 → AI 理解偏差 → 输出不符合预期
- 写得清晰 → AI 理解准确 → 输出质量高

## 2. 为什么 Prompt 这么重要？

**同一个模型，不同 prompt，输出天差地别。**

### 示例：让 AI 分析食物热量

**差的 prompt：**
```
用户：苹果
AI：苹果是一种水果，富含维生素 C 和膳食纤维...（答非所问）
```

**好的 prompt：**
```
System：你是营养师，用户输入食物名称，你回复热量（单位：千卡/100g）
用户：苹果
AI：52 千卡/100g
```

差别在哪？
- 差的 prompt：没说要干什么，AI 只能猜
- 好的 prompt：明确角色、任务、输出格式

## 3. Prompt 的组成部分

一个完整的 prompt 通常包含：

```
┌─────────────────────────────────────┐
│ System Prompt（系统提示）            │  ← 设定角色、规则、风格
├─────────────────────────────────────┤
│ Context（上下文）                    │  ← 背景信息、相关知识
├─────────────────────────────────────┤
│ Examples（示例）                     │  ← Few-shot：给几个例子
├─────────────────────────────────────┤
│ User Input（用户输入）               │  ← 具体任务
├─────────────────────────────────────┤
│ Output Format（输出格式）            │  ← 要求输出的结构
└─────────────────────────────────────┘
```

类比 Java 方法：
```java
/**
 * System Prompt = 方法注释（说明这个方法是干什么的）
 * Context = 方法参数（提供必要信息）
 * Examples = 单元测试（展示预期行为）
 * User Input = 实际调用时传的参数
 * Output Format = 返回值类型
 */
public NutritionInfo analyze(String food) { ... }
```

## 4. System Prompt vs User Prompt

### System Prompt（系统提示）

**作用**：设定 AI 的"人设"和行为规则

```python
system_prompt = """
你是一个专业的营养师，名叫小健。
你的职责是：
1. 分析食物的营养成分
2. 给出健康饮食建议
3. 回答要简洁、科学、友好

你不能：
1. 提供医疗诊断
2. 推荐药物
3. 给出绝对化的承诺
"""
```

**特点**：
- 全局生效，对整个对话都有效
- 用户看不到（通常）
- 类比 Java 的类级别配置

### User Prompt（用户提示）

**作用**：具体的任务和输入

```python
user_prompt = "我今天早餐吃了两个鸡蛋和一杯牛奶，摄入了多少蛋白质？"
```

**特点**：
- 每次对话都可能不同
- 用户直接输入的内容
- 类比 Java 的方法参数

### 职责分配原则

| 内容 | 放在哪 |
|------|--------|
| 角色定位 | System |
| 行为规则 | System |
| 输出格式要求 | System（通用格式）或 User（特定任务） |
| 背景知识 | System 或 Context |
| 具体任务 | User |
| 用户数据 | User |

## 5. 编写高质量 Prompt 的原则

### 原则 1：清晰具体

**差：**
```
帮我分析一下这个
```

**好：**
```
分析"红烧肉"的营养成分，包括热量、蛋白质、脂肪、碳水化合物，单位用克和千卡
```

### 原则 2：提供上下文

**差：**
```
这样吃健康吗？
```

**好：**
```
我是 30 岁男性，轻度运动，目标是减脂。
今天摄入：早餐鸡蛋 2 个 + 牛奶 250ml，午餐米饭 200g + 鸡胸肉 150g。
这样的饮食结构健康吗？
```

### 原则 3：指定输出格式

**差：**
```
告诉我苹果的营养
```

**好：**
```
用 JSON 格式返回苹果的营养信息：
{
  "name": "苹果",
  "calories": 52,
  "protein": 0.3,
  "fat": 0.2,
  "carbs": 14
}
```

### 原则 4：给出示例（Few-shot）

**Zero-shot（不给例子）：**
```
提取食物名称和数量
输入：我吃了两个苹果
```

**Few-shot（给例子）：**
```
提取食物名称和数量，格式：食物名|数量

示例 1：
输入：我吃了两个苹果
输出：苹果|2 个

示例 2：
输入：喝了一杯牛奶
输出：牛奶|1 杯

现在处理：
输入：吃了三片面包
```

Few-shot 能显著提升准确率，尤其是格式化任务。

### 原则 5：分步骤思考（Chain of Thought）

**直接问：**
```
用户：我今天吃了炸鸡，热量超标了吗？
AI：可能超标了（不够准确）
```

**引导思考：**
```
用户：我今天吃了炸鸡，热量超标了吗？请按以下步骤分析：
1. 估算炸鸡的热量
2. 我的每日热量需求是多少（30 岁男性，轻度运动）
3. 对比判断是否超标

AI：
1. 炸鸡约 100g 含 250 千卡，假设你吃了 200g，约 500 千卡
2. 30 岁男性轻度运动，每日需求约 2200-2500 千卡
3. 500 千卡占每日需求的 20-23%，如果这是一餐，不算超标
```

CoT 让 AI 展示推理过程，输出更可靠。

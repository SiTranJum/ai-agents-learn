# 课程 02：Prompt Engineering（提示词工程）

## 学习目标
- 理解什么是 Prompt Engineering，为什么它很重要
- 掌握编写高质量 prompt 的核心原则
- 学会使用 System Prompt 和 User Prompt
- 理解 Few-shot Learning（少样本学习）
- 掌握常见的 prompt 模式和技巧
- 能为健康管家 Agent 设计有效的 prompt

## 知识点
1. **Prompt Engineering 基础**
   - 什么是 prompt
   - 为什么 prompt 质量直接影响输出质量
   - Prompt 的组成部分

2. **System Prompt vs User Prompt**
   - System：设定角色、行为规则、输出格式
   - User：具体任务和输入
   - 如何合理分配职责

3. **编写高质量 Prompt 的原则**
   - 清晰具体
   - 提供上下文
   - 指定输出格式
   - 给出示例（Few-shot）
   - 分步骤思考（Chain of Thought）

4. **常见 Prompt 模式**
   - 角色扮演
   - 结构化输出（JSON、表格）
   - 思维链（CoT）
   - 自我一致性
   - ReAct（推理 + 行动）

5. **针对健康管家的 Prompt 设计**
   - 如何让 AI 理解营养学知识
   - 如何控制回答的专业性和友好度
   - 如何处理敏感健康问题

## 实践任务
1. 对比不同 prompt 的输出质量
2. 编写健康顾问的 System Prompt
3. 实现 Few-shot 营养分析
4. 使用 CoT 让 AI 解释推理过程
5. 设计结构化输出（JSON 格式）

## 预计时长
- 讲解：40 分钟
- 实践：40 分钟
- 总计：1.5 小时

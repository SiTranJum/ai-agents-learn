# 课程 14 学习笔记

## 核心概念

### 三类错误
- 工具错误（API 超时）：try-except 搞定
- LLM 输出错误（格式不对、幻觉）：不会抛异常，需要校验器
- 业务逻辑错误（建议不合理）：不会抛异常，需要业务规则

### 自我修正 vs 简单重试
- retry（课程 13）：同样的输入再来一次，适合网络抖动
- 自我修正（本课）：带着错误信息重试，适合 LLM 输出有问题
- 关键区别：自我修正告诉 LLM "你哪里错了"，不是盲目再来

### Guardrails（护栏）
- 输入护栏：过滤危险输入（医疗诊断、极端饮食）
- 输出护栏：拦截不安全建议（断食、催吐）
- 业务护栏：确保数据在合理范围内（热量不低于 1200 卡）
- 健康场景护栏是必须的，不是可选的

### 完整流程
```
用户输入 → [输入护栏] → LLM 生成 → [校验器+自我修正] → [输出护栏] → 返回
```

# 课程 14：错误处理与自我修正机制

## 学习目标

1. 理解 Agent 中错误的分类（工具错误、LLM 错误、业务错误）
2. 掌握自我修正（Self-Correction）的原理和实现
3. 学会设计 Guardrails（护栏）防止 Agent 失控
4. 实现一个带自我修正能力的健康管家 Agent

## 知识点

### 1. Agent 中的错误分类
- 工具执行错误（API 超时、数据库异常）
- LLM 输出错误（格式错误、幻觉、逻辑错误）
- 业务逻辑错误（不合理的建议、违反约束）

### 2. 自我修正（Self-Correction）
- 什么是自我修正：Agent 发现自己的错误并主动修复
- 实现方式：输出校验 → 错误反馈 → 重新生成
- 与简单重试的区别：重试是盲目再来，自我修正是带着错误信息再来

### 3. Guardrails（护栏）
- 输入护栏：过滤不合理的用户输入
- 输出护栏：校验 LLM 输出是否合规
- 业务护栏：确保建议在安全范围内（健康场景尤其重要）

### 4. 实践案例
- 健康场景：饮食记录解析的自我修正
- 输出校验 + 错误反馈 + 重新生成

## 预计时长

60-90 分钟

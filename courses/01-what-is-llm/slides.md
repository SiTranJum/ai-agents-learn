# 课件：什么是 LLM？

## 1. LLM 的定义

**LLM = Large Language Model（大语言模型）**

简单来说，LLM 是一个"超级聪明的文字接龙程序"：
- 你给它一段文字
- 它预测接下来最可能出现的文字
- 不断重复，直到生成完整回答

但它不是简单的接龙，而是：
- 训练了海量文本（整个互联网级别）
- 学会了语言规律、知识、推理能力
- 能理解上下文、完成复杂任务

## 2. LLM 能做什么？

| 能力 | 示例 |
|------|------|
| 文本生成 | 写文章、写代码、写邮件 |
| 问答 | 回答问题、解释概念 |
| 翻译 | 中英互译、多语言翻译 |
| 总结 | 长文总结、提取要点 |
| 分析 | 情感分析、意图识别 |
| 推理 | 逻辑推理、数学计算 |

**关键点**：LLM 不是搜索引擎，它是基于训练数据"生成"答案，而不是"检索"答案。

## 3. LLM 的工作原理（简化版）

### 3.1 Token 化
```
输入文本："你好，世界"
↓
Token 化：["你好", "，", "世界"]
↓
转换为数字：[123, 45, 678]
```

- **Token** 是 LLM 处理的最小单位
- 英文：1 个单词 ≈ 1-2 tokens
- 中文：1 个字 ≈ 1.5-2 tokens
- 标点、空格也算 token

### 3.2 预测下一个 Token
```
输入："今天天气"
↓
LLM 计算概率分布：
  "很" → 30%
  "真" → 25%
  "不" → 20%
  "怎么样" → 15%
  ...
↓
根据 temperature 参数选择一个
```

### 3.3 逐步生成
```
"今天天气" → "很" → "好" → "，" → "适合" → "出门" → "。"
```

每次生成一个 token，直到：
- 达到 max_tokens 限制
- 生成结束符号
- 用户停止

## 4. 关键参数详解

### 4.1 Temperature（温度）
控制输出的随机性：

| 值 | 效果 | 适用场景 |
|----|------|---------|
| 0 | 完全确定，每次输出相同 | 数据提取、分类 |
| 0.3-0.7 | 稳定但有变化 | 问答、翻译 |
| 0.8-1.0 | 创造性强 | 写作、头脑风暴 |
| >1.0 | 非常随机 | 实验性场景 |

### 4.2 Max Tokens
限制输出长度：
- 包括输入 + 输出的总 token 数
- 超过限制会被截断
- 影响成本（按 token 计费）

### 4.3 System Prompt
设定 AI 的角色和行为：
```python
system_prompt = "你是一个专业的健康顾问，回答要简洁、科学。"
```

### 4.4 User Prompt
用户的实际输入：
```python
user_prompt = "如何保持健康的饮食习惯？"
```

## 5. API 调用流程

```
1. 准备 API Key
   ↓
2. 构造请求
   - model: 选择模型
   - messages: 对话历史
   - temperature: 随机性
   - max_tokens: 长度限制
   ↓
3. 发送请求
   ↓
4. 接收响应
   - content: 生成的文本
   - usage: token 使用量
   - finish_reason: 结束原因
```

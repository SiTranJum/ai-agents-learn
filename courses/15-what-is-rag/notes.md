# 课程 15 学习笔记

## 核心概念

### LLM 的三个知识盲区
- 时效性知识：训练数据有截止日期
- 私有知识：从未见过你的数据
- 精确数据：会估算但不一定准确，容易幻觉

### RAG 的本质
- RAG = 开卷考试（先查资料再回答）
- 本质就是：调用 LLM 之前，先检索相关知识塞进 Prompt
- 比微调更灵活、成本更低、更新更快

### RAG 完整流程
```
离线：文档 → 分块(Chunking) → Embedding → 存入向量数据库
在线：用户提问 → Embedding → 向量检索 → 拼入 Prompt → LLM 回答
```

### Embedding（向量化）
- 把文本变成数字向量（如 1536 维）
- 语义相近的文本，向量距离近
- 解决传统关键词搜索的局限（语义相关但用词不同的内容搜不到）

### 向量数据库
- 存储向量，擅长快速找到最相似的向量
- 常见选择：Milvus（开源）、Pinecone（云服务）、Chroma（轻量）、pgvector（PG 插件）
- 健康管家项目用 Milvus

### 健康管家的 RAG 场景
- 食物营养知识库（嘌呤、热量、GI 值等权威数据）
- 健康建议知识库（营养学指南、饮食建议）
- 用户记忆库（过敏信息、健康目标、饮食偏好）

### Java 类比
- 知识库 = 数据库参考数据表
- Embedding = 把对象序列化成特征向量
- 向量检索 = `SELECT ... ORDER BY vector_distance LIMIT 3`
- RAG = Service 层先查数据库，再调 LLM

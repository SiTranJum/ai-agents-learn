## 阶段二：基础知识课程

> 目标：建立 AI Agent 的完整认知框架，掌握核心基础能力
>
> 针对用户调整：5 年后端经验，概念课讲快些，重点放在动手实践

### 2.1 AI 与 LLM 基础
- 课程 01：什么是 LLM？核心原理与能力边界 ✅
  - 实际内容：LLM 原理、Token/Temperature/上下文窗口、OpenAI SDK、多轮对话、Transformer、模型参数
  - 已覆盖原计划 03（环境搭建 + 调用 API）和 04（核心参数）的内容
- 课程 02：Prompt Engineering — 如何跟 AI 有效沟通 ✅
  - 实际内容：Prompt 组成、System/User Prompt、五大原则（清晰具体、上下文、输出格式、Few-shot、CoT）
- 课程 03：Tool Use（工具调用）— Agent 的手和脚 ✅
  - 实际内容：Function Calling 原理、工具定义（JSON Schema）、多轮调用流程、用户体验设计

### 2.2 AI Agent 核心概念
- 课程 04：什么是 AI Agent？与普通 LLM 调用的本质区别
- 课程 05：Agent 的核心循环 — 感知、思考、行动（类比 Spring MVC 的请求处理流程）
- 课程 06：Memory（记忆系统）— 短期记忆与长期记忆
- 课程 07：Planning（规划能力）— Agent 如何拆解复杂任务

### 2.3 基础实践
- 课程 08：动手做一个最简 Agent（对话 + 工具调用）
- 课程 09：给 Agent 加上记忆能力
- 课程 10：基础阶段总复习 + 小测验

> 目标：综合运用所学，完成"健康管家 AI Agent"产品
>
> 产品定位：AI-first 的个人健康管理产品，对话式交互为主

> 项目代码：`health-agent/`（docs 文档与 backend/frontend 代码分离）

### 4.1 文档驱动开发流程

**Phase 1：需求文档（PRD）**
- 与用户头脑风暴，完善总 PRD（00-master-prd.md）
- 拆分子 PRD（核心对话、饮食记录、记忆系统、RAG 知识库等）
- 明确功能优先级、用户故事、交互流程

**Phase 2：技术方案（Spec）**
- 编写总体架构 Spec（00-architecture.md）

**Phase 3：开发实施**
- 按 MVP Phase 1/2/3 分期开发
- 每个功能模块：设计 → 开发 → 测试 → 集成
- 持续迭代优化

### 4.2 MVP 分期计划

**MVP Phase 1：AI 对话 + 核心记录**
- AI 对话界面（核心入口）
- 自然语言饮食记录
- 食物营养查询
- 身体数据记录与可视化
- 个人健康档案（基础版）
- AI 基础建议能力

**MVP Phase 2：AI 记忆 + 主动能力**
- 用户习惯记忆系统
- 拍照识别食物
- AI 方案制定（对话式协商）
- AI 数据分析与趋势建议
- AI 主动提醒
- 辅助记录（睡眠、饮水、运动）

**MVP Phase 3：深度个性化 + 完整体验**
- AI 周报/月报自动生成
- 记忆驱动的个性化推荐
- 条形码扫描
- 生理期、排便等辅助记录
- 数据导出
- 计划执行追踪与动态调整

### 4.3 项目收尾
- 健康领域的 AI 安全与合规 — 免责声明、数据隐私
- 性能优化与可观测性 — 监控、日志、错误追踪
- 部署方案
- 学习总结与复盘

## 计划说明

1. **课程内容会根据学习进度调整**：已有基础的部分可跳过或合并
2. **每个课程包含**：知识讲解 → 代码实践 → 互动问答 → 小结
3. **节奏灵活**：随时可以加速、放慢、跳过或深入某个话题
4. **类比教学**：尽量用 Java/Spring 生态的概念做类比，降低理解成本
5. **项目驱动**：基础课程的实践案例会尽量贴近健康场景，为阶段四做铺垫
6. **本文档持续更新**：随着学习推进和需求变化，计划会相应调整

## 课程文件组织规范

每个课程按以下结构组织在 `courses/` 目录下：

```
courses/
├── 01-what-is-llm/              # 课程文件夹（编号-英文名）
│   ├── plan.md                  # 课程详细计划
│   ├── slides.md                # 课件内容
│   ├── code/                    # 代码示例和练习
│   ├── notes.md                 # 学习笔记
│   └── summary.md               # 课程小结
├── 02-prompt-engineering/
│   └── ...
```

**执行流程：**
1. 课程开始前 → 创建文件夹，生成 `plan.md` 和 `slides.md`
2. 课程进行中 → 代码写入 `code/`，笔记记录到 `notes.md`
3. 课程结束后 → 生成 `summary.md` 总结关键收获

详细规范见 `CLAUDE.md` 的"课程文件组织规范"章节。
